## Findings

### Finding 1 — SITE-07 partial: stale bundle files on disk [hygiene]

**Severity:** LOW (hygiene)
**Affected REQ-ID:** SITE-07

Three legacy bundle files are present on disk:
- `public/data/center-bonn.json` (99,043 lines)
- `public/data/center-muenchen.json` (105,470 lines)
- `public/data/center-muenster.json` (87,580 lines)

Origin: commit `17babca` (Phase 7-03) intentionally removed these. Commit `20611e0` (2026-04-15 Phase 8 rescope) restored them as "salvaged synthetic data" from the reverted 08-03 attempt.

**Runtime impact:** NONE. Production load path is `server/fhirApi.ts` → `manifest.json`, which lists only the 7 valid sites. The stale files are not served, not referenced, and not reachable.

**Acceptance-criterion conflict:** SITE-07 explicitly called for these files to be absent. The files currently exist. Strict reading makes SITE-07 partial.

**Recommended fix (post-milestone):** Delete the three files. Single chore commit.

### Finding 2 — Dead-code quartet: auditService + usePageAudit + safeJson + useLocalStorageState [hygiene]

**Severity:** LOW (hygiene)
**Affected REQ-ID:** None (all pre-existing, none depend on these files)

Four files exist but are not called anywhere in the render tree:
- `src/services/auditService.ts` (89 lines) — has a **pre-existing TS build error**: `Module '"../config/clinicalThresholds"' has no exported member 'MAX_AUDIT_ENTRIES'` (line 40)
- `src/hooks/usePageAudit.ts` (19 lines) — imports `auditService` but is never imported by any component
- `src/hooks/useLocalStorageState.ts` (32 lines) — never imported
- `src/utils/safeJson.ts` (12 lines) — never imported

Origin: same commit `20611e0` as Finding 1 — salvaged from the reverted 08-03 attempt.

**Runtime impact:** NONE. The Phase 9 audit beacon bypasses this module and uses plain `fetch` directly. No v1.5 requirement depends on any of the four files.

**Build impact:** `npm run build` emits one TS error (the `MAX_AUDIT_ENTRIES` import). The existing runtime is unaffected (Vite dev + the rest of `tsc -b` compiles), but it is noise for developers.

**Recommended fix (post-milestone):** Either (a) delete the four files entirely or (b) add the missing `MAX_AUDIT_ENTRIES` export to `clinicalThresholds.ts` and wire `usePageAudit` into `App.tsx`. Option (a) is cleaner — the direct `fetch` pattern in `OutcomesPage.tsx` demonstrated the module isn't needed.
