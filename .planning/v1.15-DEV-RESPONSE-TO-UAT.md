## Trajectory tooltip & clickable curves (I1 follow-up)

**You wrote:** "Highlighting improved. Pop-up still shows the closest point on the x axis rather than the highlighted point. Furthermore, the individual curves could be clickable to navigate to the respective case."

**What changed:**
- **Tooltip now follows the hovered point.** The pop-up shows the point physically under the cursor (its pseudonym + value), not the nearest point on the x-axis. (Done with an imperative tooltip so it stays cheap on dense scatter — no render lag.)
- **Individual trajectory lines are now clickable** → clicking a patient's curve opens that patient's case, alongside the scatter-point click. Same access gate (you only reach cases in your centres).

**How to verify:** Analyse → Verläufe → enable "Streupunkte" → hover different points: the pop-up shows the hovered point. Enable "Individuelle Kurven" → click a line → that patient's case opens.

**Status:** ✅ Fixed.

## "Verläufe" performance & loading status (I2 follow-up)

**You wrote:** "'wird berechnet' shows very briefly, then white cards, and the dashboard remains unresponsive until the plots are loaded… keep the UI responsive… The view is reset when leaving to another view. Can we use multithreading?"

**What changed (all three):**
- **Multithreading:** cohort aggregation now runs in a **Web Worker** (off the main thread). The UI thread stays free while the data is computed.
- **Responsive rendering:** the three panels (OD/OS/combined) mount **progressively** (one at a time, yielding between), with a clear loading status held until they're ready — no more "white cards then freeze". You can toggle Anzeigeebenen and close Einstellungen while the plots build.
- **View persists across navigation:** your layer choices, metric and axis settings are now remembered, so leaving Analyse and returning restores the view instead of resetting it.

**How to verify:** Analyse → Verläufe on a large cohort (e.g. all 245) → loads with a status indicator, no frozen white cards; toggling a layer / closing Einstellungen stays responsive. Enable some layers, navigate away and back → they're restored.

**Status:** ✅ Fixed. (A foreground glance to confirm the feel on your machine is welcome — for very large enabled scatter clouds, moving that one case to canvas is a noted future option.)

## Cohort overlay (I3 follow-up + "implement the backlog")

**You wrote:** "IQR shading appears in legend as if it was an additional line… explain it." + (Further Remarks) "Implement the backlog points on the cohort overlay."

**What changed:**
- **IQR legend** entry now renders as a **shaded box swatch** (a range), not a line, so it reads correctly.
- **Relative-time aggregation + axis (backlog item):** the case trajectory X-axis is now **"Monate seit Erstvisite"** (months since the patient's first visit). The cohort is aggregated by **relative time since each peer's own first visit**, so the patient is compared to peers at the same elapsed treatment time (the clinically meaningful comparison you asked for). IVI markers and the date highlight are placed on the relative axis; the tooltip still shows the calendar date too.
- **Overlay on the other plots (backlog item):** the cohort reference now also appears on the **Change-from-Baseline** chart (cohort median change + IQR), the **Visus & CRT distributions** (cohort distribution shown as a distinct grey reference bar on its own axis), and the **Visus-vs-CRT scatter** (cohort cloud behind your points). All gated on the "Kohorten-Referenz anzeigen" toggle.

**How to verify:** open a case → X-axis reads "Monate seit Erstvisite" (0, 7, 14 … Mon) with IVI markers; tick "Kohorten-Referenz anzeigen" → median + IQR band on the trajectory, "Kohorten-Median Änderung" + IQR on Change-from-Baseline, grey "Kohorten-Verteilung" bars on both histograms, and a "Kohorte" cloud behind the scatter.

**Status:** ✅ Fixed (legend swatch, relative-time axis, overlay on all plots).

## DocQuality main/centre parity (I5 follow-up)

**You wrote:** "Plausibility ranges… displayed in the centre-specific view but not the main view → add a collapsible table for that in the main view. Metric cards on the centre view should be equal to the main view: absolute numbers + tooltips are missing."

**What changed:** the main Dokumentationsqualität view now has a **collapsible "Plausibilitätsbereiche" table** (the same global ranges shown in the centre detail — and both now read the live settings, fixing a stale hardcoded copy). The per-centre metric cards now show the **absolute counts + the same definition tooltips** as the main view.

**How to verify:** Dokumentationsqualität (main) → expand "Plausibilitätsbereiche" → Visus/CRT/IOP ranges. Select a centre → the four cards show counts (e.g. "45 Patienten") + ℹ tooltips, matching the main view.

**Status:** ✅ Fixed.

## Quality review: tag column + remove "Behoben" (I6 follow-up)

**You wrote:** "The tags next to the table should be in their own column and should not change the table size; otherwise buttons move… Hide the 'Behoben' option, we are not planning to access the Quellsystem."

**What changed:** the status/annotation tags now live in a **fixed-width column**, so toggling a status no longer reflows the row or shifts the action buttons. The **"Behoben" action has been removed** (you confirm correct values or report errors; the "im Quellsystem korrigiert" action is gone). The annotation-persists-after-confirm behaviour from v1.14 is unchanged.

**How to verify:** Datenqualität → open a case → only **Bestätigen** and **Fehler melden** actions per row; changing a status doesn't move the buttons.

**Status:** ✅ Fixed.

## Split → Verläufe selection (Further Remarks, new)

**You wrote:** "When splitting a cohort, in the Verläufe tab the parent cohort is pre-selected and cannot be un-selected. I want to view the sub-cohorts by default and optionally add the parent."

**What changed:** after "Kohorte aufteilen", compare mode now opens with the **generated sub-cohorts pre-selected** and the **parent optional** — nothing is locked, so you can deselect any sub-cohort and add the parent if you want. (If a split produces more than the 4-cohort compare limit, the first four are pre-selected; the rest are still created and can be added manually.)

**How to verify:** Kohortenbildung → split a cohort → lands on Analyse/Verläufe with the sub-cohorts checked and the parent unchecked but addable; you can toggle any of them freely.

**Status:** ✅ Fixed.
