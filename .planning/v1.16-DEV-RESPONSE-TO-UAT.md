## Trajectory tooltips — now a single, layer-aware tooltip (J1 follow-up)

**You wrote:** "Now there are TWO tooltips — one for the nearest point on the x-axis and one for the scatter point I hover over… show the scatter tooltip only when scatter points are activated; when individual lines are activated, highlight the line on hover; when scatter tooltips are not activated → tooltip when hovering over a line; remove the x-axis related tooltips. Also: the curve-settings button is not self-explanatory."

**What changed:**
- **The x-axis tooltip is gone.** The shared "nearest point on the axis" pop-up has been removed entirely — there is now exactly **one** tooltip on the chart.
- **Layer-aware behaviour, scatter takes precedence:**
  - **Streupunkte on** → hovering a point shows that point's tooltip (pseudonym + value), and only that.
  - **Individuelle Kurven on, Streupunkte off** → hovering a line **highlights that line** and shows the tooltip for that patient.
  - When both are on, the scatter point wins (so you're never shown two things for one cursor position).
- **The settings button is now self-explanatory** — relabelled **"Diagramm-Anzeige"** so it's clear it controls which layers are drawn.

**How to verify:** Analyse → Verläufe → enable only "Streupunkte" → hover points: one tooltip, no axis pop-up. Turn Streupunkte off and "Individuelle Kurven" on → hover a line: the line is emphasised and its patient tooltip appears. The layer button reads "Diagramm-Anzeige".

**Status:** ✅ Fixed.

## "Verläufe" performance — the scatter is now drawn on a canvas (J2 follow-up, HIGH)

**You wrote:** "Performance did not improve significantly. Dashboard still unresponsive while loading plots when changing settings."

**What changed — this is the substantive fix you've been waiting on across rounds:** the previous work moved *computation* off the main thread, but the remaining cost was **Recharts building thousands of SVG nodes** for the scatter (≤1,500 points × multiple panels) every time you changed a setting — and a worker cannot help with rendering. So we changed the **rendering layer**:
- **The heavy scatter cloud is now painted on a `<canvas>`** instead of thousands of individual SVG circles. One draw call replaces the SVG-node wall, so toggling a layer / changing a setting no longer stalls the UI. Hover, the tooltip, and click-to-open-case all still work — they run against the canvas via a hit-test, so behaviour is identical to before, just without the lag.
- Everything degrades gracefully: where canvas isn't available the old SVG path is used unchanged.

**How to verify:** Analyse → Verläufe on a large cohort (e.g. all 245) → enable "Streupunkte" and toggle layers / change settings repeatedly: the UI stays responsive (no white-card freeze). Hovering and clicking points behaves exactly as before.

**Status:** ✅ Fixed (canvas rendering for the scatter).

## "Kohorten vergleichen" menu froze / was unusable (NEW regression, HIGH)

**You wrote:** "Default works. The 'Kohorten vergleichen' menu freezes now and is not usable."

**What changed:** opening the compare drawer rendered *N* cohorts × series and ran the aggregation for **each** cohort synchronously on the main thread — so it multiplied the same bottleneck by the cohort count and locked the UI. Now the **per-cohort aggregation runs off the main thread** (one worker request per selected cohort, assembled when all complete), and the multi-cohort panels mount progressively. The drawer opens and stays interactive while the comparison builds.

**How to verify:** Kohortenbildung → build/split into several cohorts → Analyse → "Kohorten vergleichen" → the menu opens responsively, you can change the selection, and the plots fill in without freezing.

**Status:** ✅ Fixed.

## Cohort overlay follow-ups (J3 follow-up)

**You wrote:** "IQR legend missing for cohort median CRT. Relative timestamps for cohort overlay works. Potentially add overlay for IOD (distribution / aggregated line). Use non-relative timestamps for the individual view. In Behandlungsverlauf the injections are not clickable anymore (highlight injections temporarily in the plot like observations; if feasible, highlight in the Intravitreale Injektionen list)."

**What changed (all four):**
- **CRT IQR legend (K3a):** the cohort CRT median band now has its own shaded-box legend swatch, matching the Visus band — the IQR range reads as a range, not a line.
- **Individual axis back to calendar dates (K3c):** following your clarification ("in der patientenindividuellen Ansicht ist sonst alles mit konkretem Datum, im Vergleich macht nur das relative Datum Sinn"), the **single-patient trajectory X-axis is back to calendar dates**. The **cohort overlay still aggregates peers by relative time** since each peer's own first visit (the clinically meaningful comparison you confirmed "works"), mapped onto the patient's calendar dates. IVI markers and the highlight follow the calendar axis again.
- **IOP/IOD overlay (K3b):** the cohort reference now also appears on the **Augeninnendruck (IOD)** plot — a cohort median line with IQR band, aggregated by the same relative-time alignment, on a fixed 0–30 mmHg scale.
- **Injections clickable again (K3d):** in the Behandlungsverlauf, **clicking an injection temporarily highlights it** in the plot (like an observation) **and** highlights the matching entry in the "Intravitreale Injektionen" list. Clicking again clears it; navigating to another case clears it too.

**How to verify:** open a case → X-axis shows calendar dates; tick "Kohorten-Referenz anzeigen" → Visus **and** CRT median + IQR bands each have a legend swatch, and the IOD plot shows a cohort median + IQR. Click an injection in the Intravitreale-Injektionen list → it's emphasised in the plot and the list; click another case → the highlight resets.

**Status:** ✅ Fixed (a/b/c/d).
