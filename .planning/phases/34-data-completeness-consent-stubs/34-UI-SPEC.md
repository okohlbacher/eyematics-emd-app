## Typography

This contract covers only the typographic roles used inside the NEW Datenvollzähligkeit card component. The page `h1` (28px) is a pre-existing LandingPage element, unmodified in this phase, and is not part of this contract.

| Role | Size | Weight | Line Height | Notes |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Subtitles, paragraph text (`text-[var(--color-ink-2)]`) |
| Label | 12px | 400 | 1.4 | Secondary labels, sub-text (`text-[var(--color-ink-2)]`) |
| Caption | 11px | 600 | 1.3 | ALL CAPS section headers, uppercase tracking `tracking-[0.12em]` (`text-[var(--color-ink-3)]`) |
| Display | 32px | 600 | 1.0 | KPI metric numbers (font-data / JetBrains Mono, `tracking-[-0.03em]`) |

Rules:
- Body (14px / 400 / 1.5): all non-headline content in cards and rows
- Caption (11px / 600 / uppercase): section header labels above sub-tiles (e.g. "DATENVOLLZÄHLIGKEIT")
- Display (32px / 600 / mono): the primary numeric value in the card (the fraction as a percentage)
- Secondary metric values (consented count, total count) use Label (12px / 400 / 1.4)
- No new font sizes or weights outside this table
- Pre-existing page h1 (28px / 600 / 1.15 / `tracking-[-0.02em]`) is unchanged and not governed by this contract

Source: `src/pages/LandingPage.tsx`, `src/index.css`

**Accent reserved for:** the Datenvollzähligkeit card icon container background (`teal-soft`) and icon stroke (`teal`), matching the existing KPI card pattern where each card has one tone pair.

Dark mode: all tokens have `.dark` overrides in `src/index.css` — no hard-coded hex values allowed in new markup.

## Card Layout: Datenvollzähligkeit (LandingPage)

### Position

Inserted as the **5th card** in the existing `grid-cols-4` KPI row on LandingPage, making it a **5th column spanning full width below**, OR integrated as a **separate full-width row** beneath the 4 KPI tiles. Recommended: **full-width row** placed between the KPI tile row and the Centers + Right-rail row, spanning all columns (`col-span-4` or inside its own `px-8` div with `mb-3.5`). This avoids breaking the symmetric 4-column KPI layout.

### Card Internal Structure

```
[Tile]
  ┌ Icon container (w-8 h-8, rounded-lg, teal-soft bg) — ShieldCheck icon (teal)
  │
  ├ Fraction display: "21 %" — 32px semibold mono (color: semantic, see above)
  │
  ├ Label row:
  │   left:  "Datenvollzähligkeit" — 12px ink-2
  │   right: "N / M Patienten" — 11px ink-3 (consented / total)
  │
  └ Progress bar: h-1.5 rounded-full, teal-soft track, semantic color fill (% of fraction)
```

Icon: `ShieldCheck` from lucide-react (conveys consent/compliance). Size: `w-4 h-4`.

Note: The `Tile` component's internal padding is inherited as-is from `src/components/primitives/Tile.tsx` — no new padding values are introduced.

### Responsive behavior

Follows the same `grid-cols-1 md:grid-cols-4` pattern as the existing KPI tiles row. On mobile the completeness row stacks below the 4 tiles.

## States and Interactions

### Loading state
- While `loading === true` in DataContext, the card is **not rendered** — same guard as the existing stats cards. No skeleton needed; the existing full-page loader covers it.

### No-data state (all stubs, zero consented patients)
- Card renders with fraction = 0 %, coral fill on progress bar.
- Primary metric shows "0 %".
- Sub-label shows "0 / N Patienten".
- Caption still reads "DATENVOLLZÄHLIGKEIT".
- This state should not occur with valid data but must not crash.

### Site filter active (per-user restriction or filter selection)
- Card values update reactively — numerator and denominator both filtered to the current site set (D-10 from CONTEXT.md).
- No special visual treatment needed for filtered state; the values just update.

### Error state
- If bundle load fails, DataContext already shows a global error state. The card does not need its own error boundary — it inherits the existing LandingPage loading/error guard.

## Accessibility

- `aria-label` on the progress bar element: use `datenvollstaendigkeitAriaLabel` i18n key with the computed percentage interpolated.
- The metric value `{N} %` must have sufficient contrast against `var(--color-surface)` background. Teal, sage, amber, and coral tokens are already validated for WCAG AA in Phase 14 (`ARIA chart labels` + Phase 17 `DARK_EYE_COLORS WCAG 4.5:1`).
- The card is read-only (no interactive elements). No keyboard focus / tabIndex management needed.
- Icon (`ShieldCheck`) is decorative — `aria-hidden="true"`.
