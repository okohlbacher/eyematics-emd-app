## Interaction Contracts

### Theme Toggle (D-08, D-09)

- **Location:** Sidebar footer, inserted between the language switcher row (`Layout.tsx:90-97`) and the user/logout row (`Layout.tsx:100-114`).
- **Component:** Single icon button `<ThemeToggle />`. Cycles: Sun (Light) → Moon (Dark) → Monitor (System) → Sun.
- **Icon map:** Sun = `<Sun className="w-3.5 h-3.5" />`, Dark = `<Moon className="w-3.5 h-3.5" />`, System = `<Monitor className="w-3.5 h-3.5" />`.
- **Button class:** Matches language switcher: `flex items-center gap-2 w-full px-2 py-1.5 text-xs text-slate-400 hover:text-white transition-colors rounded hover:bg-slate-700/50`.
- **Tooltip:** Title attribute shows the NEXT mode label (e.g. when Light is active, tooltip reads "Switch to Dark"). i18n keys: `themeLight`, `themeDark`, `themeSystem`.
- **State persistence:** `localStorage` key `emd-theme`. Values: `'light'` | `'dark'` | `'system'`.
- **FOUC prevention:** Inline `<script>` in `index.html` `<head>` before all other scripts (D-12). Reads `emd-theme`, applies `document.documentElement.classList.add('dark')` synchronously.

### Audit Filter Panel (D-01 through D-04)

The existing filter panel (`showFilters` toggle) is expanded from 1 control (time range) to 5 controls laid out in a single row on wide screens and stacked on narrow:

| Control | Type | Width | Notes |
|---------|------|-------|-------|
| User | `<select>` | max-w-[160px] | Populated from distinct users in fetched entries; first option "All users" |
| Action category | `<select>` | max-w-[180px] | Options: All, Auth, Data, Admin, Outcomes |
| Date range | Two `<input type="date">` | max-w-[140px] each | From / To labels above; clears to `''` when empty |
| Cohort-hash search | `<input type="search">` | flex-1 min-w-[160px] | Placeholder: "Search cohort hash…" |
| Failures only | `<input type="checkbox">` with label | auto | Filters to status >= 400 |

- Filter panel container: `bg-white rounded-xl border border-gray-200 p-4 dark:bg-gray-800 dark:border-gray-700`
- Filter control labels: `text-xs font-medium text-gray-500 uppercase mb-1 dark:text-gray-400`
- Filter input base class: `text-sm border border-gray-200 rounded-lg px-3 py-2 bg-white dark:bg-gray-700 dark:border-gray-600 dark:text-gray-100 focus:outline-none focus:ring-2 focus:ring-blue-500`
- Fetch debounce: 300ms after any filter change (D-04).
- Filter row layout: `flex flex-wrap items-end gap-3`

### ThemeContext

- Shape mirrors `LanguageContext.tsx` pattern: `createContext` + `ThemeProvider` + `useTheme()` hook.
- Export: `theme: 'light' | 'dark' | 'system'`, `setTheme: (t: Theme) => void`, `effectiveTheme: 'light' | 'dark'` (resolved from system preference).
- `ThemeProvider` is added to `App.tsx` wrapping the same level as `LanguageProvider`.
- `useTheme()` is consumed by `OutcomesPanel.tsx` to select `EYE_COLORS` vs `DARK_EYE_COLORS`.

## Tailwind Dark Mode Configuration

Tailwind v4 is used via `@tailwindcss/vite`. Dark mode must be configured as class-based. Add to `src/index.css` (the single Tailwind entry point):

```css
@import "tailwindcss";

/* Enable class-based dark mode */
@custom-variant dark (&:where(.dark, .dark *));
```

This enables all `dark:` utility classes to activate when `<html class="dark">` is set. Verify against Tailwind v4 docs — the `@custom-variant` directive is the v4 equivalent of `darkMode: 'class'` in v3 config.
