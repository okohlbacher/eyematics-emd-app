**Bug:** `auditMiddleware` reads `req.body` at `res.on('finish')` time (line 114), but `req.body` is only populated by `express.json()` which is scoped to `/api/auth` (server/index.ts line 144). For `POST /api/issues` and `PUT /api/settings`, the body is consumed by `readBody()` in the route handler, so `req.body` is undefined and audit entries get `body: null`.

**Fix — Part A (server/utils.ts):** Modify `readBody()` to attach the parsed body string to the request object after stream consumption. In the `req.on('end', ...)` callback, before calling `resolve(data)`, add:

```typescript
(req as unknown as Record<string, unknown>)._capturedBody = data;
```

The full `readBody` function becomes:
```typescript
export function readBody(req: import('http').IncomingMessage, maxSize = MAX_BODY_SIZE): Promise<string> {
  return new Promise((resolve, reject) => {
    let data = '';
    let size = 0;
    req.on('data', (chunk: Buffer) => {
      size += chunk.length;
      if (size > maxSize) {
        req.destroy();
        reject(new Error(`Request body too large (max ${maxSize} bytes)`));
        return;
      }
      data += chunk.toString();
    });
    req.on('end', () => {
      // Attach raw body to req for audit middleware capture (Bug 1 fix)
      (req as unknown as Record<string, unknown>)._capturedBody = data;
      resolve(data);
    });
    req.on('error', reject);
  });
}
```

**Fix — Part B (server/auditMiddleware.ts):** In the `res.on('finish')` callback, change the body capture logic (currently line 114-115) to fall back to `_capturedBody` when `req.body` is not populated:

Replace:
```typescript
const bodyStr =
  req.method !== 'GET' ? redactBody(urlPath, req.body) : null;
```

With:
```typescript
const capturedBody = (req as unknown as Record<string, unknown>)._capturedBody as string | undefined;
const rawBody = req.method !== 'GET'
  ? (req.body !== undefined && req.body !== null
      ? req.body
      : (capturedBody ? tryParseJson(capturedBody) : undefined))
  : undefined;
const bodyStr = req.method !== 'GET' ? redactBody(urlPath, rawBody) : null;
```

Also add a `tryParseJson` helper function above the middleware function:
```typescript
/** Try to parse a string as JSON; return the original string if parsing fails. */
function tryParseJson(str: string): unknown {
  try {
    return JSON.parse(str);
  } catch {
    return str;
  }
}
```

This is needed because `redactBody` calls `JSON.stringify(body)` and expects an object for REDACT_PATHS. For non-JSON bodies (like YAML settings), `tryParseJson` returns the raw string, which `redactBody` handles via `String(body)` for non-object types.

**Bug:** `validateSettingsSchema()` checks `obj.twoFactorEnabled` (top-level boolean) but the canonical settings.yaml structure has `auth.twoFactorEnabled` (nested). This means a valid settings.yaml with the nested auth section would be rejected because top-level `twoFactorEnabled` is missing.

**Fix:** Rewrite `validateSettingsSchema()` and add `export` keyword. Replace the entire function (lines 26-56 of settingsApi.ts) with:

```typescript
export function validateSettingsSchema(parsed: unknown): string | null {
  if (parsed === null || typeof parsed !== 'object') {
    return 'Settings must be a YAML object';
  }
  const obj = parsed as Record<string, unknown>;

  // Top-level numeric fields
  if (typeof obj.therapyInterrupterDays !== 'number' || !Number.isFinite(obj.therapyInterrupterDays)) {
    return 'therapyInterrupterDays must be a number';
  }
  if (typeof obj.therapyBreakerDays !== 'number' || !Number.isFinite(obj.therapyBreakerDays)) {
    return 'therapyBreakerDays must be a number';
  }

  // dataSource section
  if (obj.dataSource === null || typeof obj.dataSource !== 'object') {
    return 'dataSource must be an object';
  }
  const ds = obj.dataSource as Record<string, unknown>;
  if (typeof ds.type !== 'string' || ds.type.length === 0) return 'dataSource.type must be a non-empty string';
  if (typeof ds.blazeUrl !== 'string' || ds.blazeUrl.length === 0) return 'dataSource.blazeUrl must be a non-empty string';

  // auth section (canonical nested structure — per D-03)
  if (obj.auth === null || typeof obj.auth !== 'object') {
    return 'auth must be an object';
  }
  const auth = obj.auth as Record<string, unknown>;
  if (typeof auth.twoFactorEnabled !== 'boolean') {
    return 'auth.twoFactorEnabled must be a boolean';
  }
  if (typeof auth.maxLoginAttempts !== 'number' || !Number.isInteger(auth.maxLoginAttempts) || auth.maxLoginAttempts < 1) {
    return 'auth.maxLoginAttempts must be a positive integer';
  }
  // otpCode is optional (defaults to '123456' in initAuth.ts)
  if (auth.otpCode !== undefined && typeof auth.otpCode !== 'string') {
    return 'auth.otpCode must be a string if provided';
  }

  return null;
}
```

- **ADDED:** `auth.twoFactorEnabled` boolean check (nested — correct path)
- **ADDED:** `auth.maxLoginAttempts` positive integer check

- **ADDED:** `export` keyword so tests can import it directly (per D-05)
