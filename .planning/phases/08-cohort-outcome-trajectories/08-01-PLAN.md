    B. Implement `computeCohortTrajectory` in `src/utils/cohortTrajectory.ts`:

    1. For each PatientCase:
       - Run `getObservationsByCode(c.observations, LOINC_VISUS)` → ascending-by-date visus observations.
       - Split by `eyeOf(obs.bodySite)`:
         - `eyeObs.od = observations with eyeOf === 'od'`
         - `eyeObs.os = observations with eyeOf === 'os'`
       - For each eye panel build a `PatientSeries`:
         - baseline = first observation's logMAR (D-05: computed from patient's full record, which `getObservationsByCode` gives us ascending)
         - if 0 obs → excluded=true, return early
         - For each observation, compute:
           - `date`, `decimal = obs.valueQuantity.value`, `logmar = decimalToLogmar(decimal)`, `snellenNum/Den = decimalToSnellen(decimal)`
           - `x = axisMode === 'days' ? daysBetween(baselineDate, obs.date) : treatmentIndexAt(c.procedures, obs.date, eye)`
           - `y = yMetric === 'absolute' ? logmar : yMetric === 'delta' ? (logmar - baseline) : baseline === 0 ? null : clamp((logmar-baseline)/baseline*100, -200, 200)`
           - clipped flag attached to measurement when clamping actually occurred
         - sort measurements by x ascending
         - Determine sparse: measurements.length >= 2 && measurements.length < Math.ceil(gridPoints/10)
       - Build combined panel: pool OD + OS measurements; patient is included iff they have ≥1 measurement in either eye (D-20).

    2. Build shared grid per panel via `buildGrid`.

    3. Per panel, compute `medianGrid`:
       - For each grid x, interpolate EACH patient's measurements-as-(x,y) series via `interpolate`.
       - Collect non-null y's across patients; if count === 0, SKIP this grid point entirely (mismatched-span rule).
       - Otherwise emit `{x, y: median, p25, p75, n}` where sort the y's once and call `percentile(sorted, 0.5|0.25|0.75)`.
       - Exclude single-measurement patients from median interpolation (D-18): patients where `measurements.length < 2` contribute null to every grid point.

    4. Build `scatterPoints` — all individual measurements as `{x, y, patientId}`. For Δ% mode, clipped outliers are still emitted as scatter points (the UI renders them as red-outlined dots per UI-SPEC; the clipped flag lives on the Measurement).

    5. summary:
       - `patientCount` = patients with ≥1 measurement in this eye (excluded === false)
       - `excludedCount` = patients with 0 measurements in this eye (cases.length - patientCount)
       - `measurementCount` = total observations across all patients for this eye

    6. Helper `daysBetween(a, b)`: `Math.floor((new Date(b).getTime() - new Date(a).getTime()) / 86400000)`.

    7. Helper `clamp(v, lo, hi)`: `Math.max(lo, Math.min(hi, v))`.

    Keep everything pure — no module-level state beyond the exported constants. No memoization inside the utility (callers use `useMemo` per D-16).

    Respect the `spreadMode` parameter:
    - `'iqr'` (default): p25 = percentile(sorted, 0.25), p75 = percentile(sorted, 0.75)
    - `'sd1'`: p25 = mean - sd, p75 = mean + sd
    - `'sd2'`: p25 = mean - 2·sd, p75 = mean + 2·sd
    Tests only cover 'iqr'; sd1/sd2 branches are future-toggleable per D-13. Implement them anyway; add a single sanity test asserting `spreadMode='sd1'` returns symmetric bands around the median for a uniform-y cohort.
