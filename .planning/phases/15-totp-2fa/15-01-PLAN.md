<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→POST /api/auth/totp/enroll | Untrusted request body; enrollToken carries server-signed secret |
| client→POST /api/auth/totp/confirm | Untrusted request body includes enrollToken + otp code |
| authenticated user→protected /api/* | verifyLocalToken must reject enrollment-purpose tokens |
| audit pipeline→on-disk audit DB | Sensitive fields (secret, token) must never persist |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Elevation of Privilege | authMiddleware verifyLocalToken | mitigate | Task 3: reject `payload.purpose === 'totp-enroll'` so an enrollment-purpose JWT cannot authenticate any protected route |
| T-15-02 | Elevation of Privilege | authMiddleware verifyKeycloakToken | mitigate | Task 3: reject `raw.purpose === 'totp-enroll'` in Keycloak branch |
| T-15-03 | Information Disclosure | auditMiddleware body capture | mitigate | Task 4: add `enrollToken` and `totpSecret` to REDACT_FIELDS so they are replaced with [REDACTED] in audit.db |
| T-15-04 | Information Disclosure | auditMiddleware path matching | mitigate | Task 4: add `/api/auth/totp/enroll` and `/api/auth/totp/confirm` to REDACT_PATHS so field redaction is invoked for those bodies |
| T-15-05 | Tampering | UserRecord schema drift | accept | Task 2: new fields are optional; older users.json records without them remain valid; migration not needed |
| T-15-06 | Denial of Service | TOTP routes outside PUBLIC_PATHS | mitigate | Task 3: add the two TOTP routes to PUBLIC_PATHS so pre-session enrollment calls reach handlers (no 401 loop) |

Residual risks: None introduced by this scaffold plan. Endpoint behaviour is added by Plans 02 and 03, which carry their own threat models.
</threat_model>
