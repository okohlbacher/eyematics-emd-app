## Spacing Scale

Phase 8 uses Tailwind's default 4px grid. Declared scale (multiples of 4):

| Token | Tailwind | Value | Usage in Phase 8 |
|-------|----------|-------|-------------------|
| xs | `1` | 4px | Icon-label gaps inside toggle rows |
| sm | `2` | 8px | Between radio/checkbox items, card-internal gaps |
| md | `4` | 16px | Card internal content gaps, drawer section spacing |
| lg | `5`/`6` | 20–24px | Card padding (`p-5` = 20px matches `AnalysisPage`), header-to-content gap |
| xl | `8` | 32px | Page padding (`p-8` matches `AnalysisPage`, `CohortBuilderPage`) |
| 2xl | `12` | 48px | Vertical break between summary-card row and chart row |

Exceptions:
- **Chart panel internal height:** fixed `height={320}` (px) on `ResponsiveContainer` — not a spacing token; a chart-rendering parameter. Matches `VisusCrtChart` (`height={300}`) with +20 for the legend band.
- **Drawer width:** `w-96` (384px) on desktop, `w-full` on viewports < 640px. 384 is divisible by 4.
- **Gear icon hit target:** `p-2` (8px) on a 16-px icon yields a 32-px square; acceptable for a desk app. No 44px touch-target requirement (clinical desktop app).

## Color

Phase 8 inherits the app's existing palette. No new hex values introduced except chart role assignments below.

### Surface palette (60 / 30 / 10)

| Role | Value | Tailwind | Usage |
|------|-------|----------|-------|
| Dominant (60%) | `#f9fafb` | `bg-gray-50` | Main page background (via `Layout.tsx`) |
| Secondary (30%) | `#ffffff` | `bg-white` | Card surfaces, drawer surface, table rows |
| Secondary chrome | `#e5e7eb` | `border-gray-200` | Card borders, drawer border, table dividers |
| Accent (10%) | `#2563eb` | `bg-blue-600` / `text-blue-600` | Primary CTA only (see reserved-for list) |
| Destructive | `#dc2626` | `bg-red-600` / `text-red-500` | Destructive confirm, "sparse — excluded" count warnings only |

**Accent (`blue-600`) reserved exclusively for:**
1. The "Download CSV" button in the Data preview panel.
2. The numeric value digits inside summary cards (e.g. "42" in `Patientenzahl: 42`) — styled `text-blue-600 font-semibold`, mirroring the `CohortBuilder` pseudonym column convention.
3. The active state of radio/checkbox inputs in the settings drawer (native `accent-blue-600`).

Accent is NOT used for: panel titles, card borders, chart strokes, drawer borders, tab bars, hover states on non-CTA buttons.

### Chart palette (assignments from `CHART_COLORS`)

Source: `src/config/clinicalThresholds.ts` — `CHART_COLORS = ['#3b82f6', '#10b981', '#f59e0b', '#ef4444', '#8b5cf6', '#ec4899']`.

| Role | Hex | Rationale |
|------|-----|-----------|
| **OD (right eye) series** | `#3b82f6` (blue-500) | Indexed 0 — convention "right eye = cool hue". Matches Ishihara-safe distinguishability against OS. |
| **OS (left eye) series** | `#f59e0b` (amber-500) | Indexed 2 — blue/amber is the canonical deuteranopia-safe pair. Avoids green/red clash. |
| **Combined series** | `#8b5cf6` (violet-500) | Indexed 4 — neutral third hue; neither eye-coded. |
| **Median overlay stroke** | same as panel series, `strokeWidth={3}` | Median is 1.5× the per-patient curve stroke so it reads as "bolder" (OUTCOME-02). |
| **Per-patient curves** | same hue as panel, `strokeOpacity={0.6}` at `strokeWidth={1.5}` | Sparse curves (D-19) drop to `strokeOpacity={0.3}`. |
| **Scatter dots** | same hue as panel, `fillOpacity={0.5}`, radius 3 | Single-measurement patients (D-18) render as radius-4 dots at `fillOpacity={0.8}` so they are not lost in the scatter cloud. |
| **Spread band (IQR)** | same hue as panel, `fillOpacity={0.15}`, no stroke | Renders as a semi-transparent area between 25th and 75th percentile polylines. 0.15 is low enough to preserve per-patient curve visibility and matches the existing `ReferenceArea` convention elsewhere in the app. |
| **Zero / baseline reference line** | `#94a3b8` (slate-400), `strokeDasharray="5 5"` | Matches `VisusCrtChart` cohort-average line style. Drawn at y=0 in Δ / Δ% modes. |
| **Clipped-outlier indicator** (Δ% mode only) | `#ef4444` (red-500) outlined dot, no fill | For values clipped at ±200% per D-10. Tooltip notes "clipped at ±200%". |

Colorblind safety: the OD/OS blue/amber pair is distinguishable under deuteranopia, protanopia, and tritanopia simulations. Violet for combined is chromatically distant from both. No red/green pairs anywhere.

### Semantic color in summary cards

| Condition | Treatment |
|-----------|-----------|
| Default numeric value | `text-blue-600` (accent) |
| Zero or excluded value | `text-gray-400` |
| Excluded-count hint | `text-xs text-gray-500` — "{N} excluded" below the main number |

### Copy rules (style guide, for executor)

- **Patients pseudo-inclusive form (DE):** always `Patient:innen` (genderstar colon form), matching `savePasswordHint` convention and aligning with BMG style guide. Never `Patienten` alone.
- **Eye labels:** `OD` / `OS` (never `rechts` / `links` in chart legends — clinical abbreviations are expected).
- **Numbers:** decimal separator follows locale — comma in DE (e.g. `Visus 0,42`), dot in EN (`Visus 0.42`). Already handled via `locale` threading in `formatDate` / chart formatters.
- **No exclamation marks** anywhere in normal status copy (mirrors existing app tone).

### Primary CTA

Primary action of this phase is **viewing**, not submitting. The only button-shaped accent-colored CTA is `outcomesPreviewExportCsv` — "CSV exportieren" / "Export CSV". All other controls are radio / checkbox / slider inputs in the drawer, which do not need a CTA label.

### Destructive actions

**None.** Phase 8 has no destructive interactions. The data preview panel does not allow deleting rows; CSV export is non-destructive; cohort membership is read-only from this view.

```
┌──────────────────────────────────────────────────────────────┐

│  <p>  {subtitle}                                              │
├──────────────────────────────────────────────────────────────┤
│  ┌──Card──┐ ┌──Card──┐ ┌──Card──┐ ┌──Card──┐                 │  ← summary cards row
│  │Patients│ │  Meas  │ │   OD   │ │   OS   │                 │     gap-4, grid-cols-4
│  └────────┘ └────────┘ └────────┘ └────────┘                 │
│                                                              │
│  ┌─OD panel──────┐ ┌─OS panel──────┐ ┌─Combined──────┐      │  ← chart row
│  │ [chart 320px] │ │ [chart 320px] │ │ [chart 320px] │      │     gap-6, grid-cols-3

│                                                              │
│  ▸ {outcomesPreviewToggleOpen}                               │  ← data preview <details>
└──────────────────────────────────────────────────────────────┘
```

### Breakpoints

Use Tailwind default breakpoints only:

| Viewport | Layout |
|----------|--------|
| `< 640px` (sm) | Summary cards stack `grid-cols-1`; chart panels stack vertically; drawer opens full-width (`w-full`). |
| `640–1023px` (md) | Summary cards `grid-cols-2`; chart panels stack vertically (viewport too narrow for 3-up comparison); drawer `w-96`. |
| `1024–1279px` (lg) | Summary cards `grid-cols-4`; chart panels stack vertically OR 2-up+1-below — **choose vertical stack** for readability consistency. |
| `≥ 1280px` (xl) | Summary cards `grid-cols-4`; chart panels `grid-cols-3` side-by-side (the target layout). |

Decision on narrow-viewport panel layout (CONTEXT.md §Discretion): **vertical stack**, not horizontal scroll. Horizontal scroll hides context and breaks the "compare OD / OS / combined at a glance" mental model; vertical stack preserves it at the cost of scroll.

### Summary card (4 instances)

```
┌────────────────────────────┐
│  <Icon>  Patient:innen     │  ← 12px label, icon 16px, gap-2
│                            │
│     42                     │  ← 30px (text-3xl) font-semibold text-blue-600
│                            │
│  3 ausgeschlossen          │  ← 12px text-gray-500 (conditional)
└────────────────────────────┘
```

- Container class: `bg-white rounded-xl border border-gray-200 p-5` (matches `AnalysisPage` card convention).
- Icons (lucide):
  - Patients: `Users`
  - Total measurements: `Activity`
  - OD measurements: `Eye` (lucide `Eye`)
  - OS measurements: `Eye` (same icon; distinction via label)
- Icon color: `text-gray-400` — decorative only, not carrying meaning.
- Value font: `text-3xl font-semibold text-blue-600` (30px). This is the only spot in Phase 8 that uses `text-3xl` — it is scoped as "card-value" and does not introduce a new general typography token.
- Number formatting: integer only for count cards; `Intl.NumberFormat(locale)` for thousand separators when count ≥ 1000.
- Live-recompute contract: values update synchronously on toggle/filter change (no spinner); recomputation is cheap because it reads the memoized aggregator.

### Chart panel (3 instances — OD, OS, Combined)

```
┌──────────────────────────────────────────┐
│  <h3> Rechtes Auge (OD)                  │  ← text-base font-semibold
│  <p>  {n} Patient:innen · {m} Messungen  │  ← text-xs text-gray-500
│                                          │
│  [ResponsiveContainer height={320}]      │
│                                          │
│  Legend                                  │  ← recharts Legend, 12px
└──────────────────────────────────────────┘
```

- Container class: `bg-white rounded-xl border border-gray-200 p-5`.
- Recharts composition: `ComposedChart` (not `LineChart` alone) to allow `Area` (spread band), `Line` (median + per-patient), and `Scatter` (individual dots + single-measurement patients) in the same viewport.
- Render order (z-stack, bottom → top):
  1. `CartesianGrid strokeDasharray="3 3"`
  2. `ReferenceLine y={0}` (Δ / Δ% modes only, stroke `#94a3b8`, dashed)
  3. Spread band `Area` (fillOpacity 0.15)
  4. Per-patient curves `Line[]` (strokeOpacity 0.6, sparse 0.3)
  5. Scatter dots
  6. Median `Line` (strokeWidth 3)
- Axis domain:
  - X: auto-scaled (Recharts default) for both "Days since baseline" and "Number of treatments" modes.
  - Y absolute mode: `[0, 2]` (logMAR clinical range).
  - Y delta mode: auto, but clamped to `[min-0.1, max+0.1]` for breathing room.
  - Y delta_percent mode: `[-200, 200]` (per D-10 clamp).
- Empty panel state: if no patients qualify (e.g. all excluded from OD per D-17), render the panel header + a centred `outcomesPanelEmpty` message in `text-sm text-gray-400`, no chart.

| Property | Value |
|----------|-------|
| Fill | same hex as panel series |
| Fill opacity | `0.15` |
| Stroke | none (`stroke="none"`) |

Median hover tooltip:
1. `{outcomesTooltipMedian}` with patient count
2. X-axis value
3. Median Y value

Scatter dot tooltip: same as per-patient curve tooltip.

Tooltip container class: `bg-white border border-gray-200 rounded-lg shadow-lg px-3 py-2 text-sm` (matches `AnalysisPage` tooltip convention).

Radio/checkbox visual: native inputs with `accent-blue-600` utility for the accent color.

### Data preview panel

Implementation: native `<details>` (honors D-27 "expandable" pattern; no JS state).

- Summary row: `<summary className="cursor-pointer text-sm font-medium text-gray-700 py-3 flex items-center gap-2">` with a chevron (`ChevronRight` / `ChevronDown`).
- Collapsed by default (D-27).
- Expanded content:
  - Header row: row count caption (`outcomesPreviewCaption`) + `Export CSV` button (accent blue, `lucide Download` icon).
  - Table: `w-full` with `thead` `bg-gray-50`, header cells `text-xs font-medium text-gray-500 uppercase`, body rows `text-sm`, dividers `divide-y divide-gray-100`. Mirrors `CohortBuilderPage` results table exactly.
  - Scroll: `max-h-96 overflow-auto` on the table wrapper (prevents the page from growing unbounded for 10k-row cohorts).

### Loading skeleton

While the aggregator runs (≥ 200ms perceived delay), render:
- Summary cards: 4 `bg-gray-100 rounded-xl h-24 animate-pulse` placeholders.
- Chart panels: 3 `bg-gray-100 rounded-xl h-96 animate-pulse` placeholders.
- No `outcomesLoading` text overlay — the pulsing placeholders carry the meaning.

Skeleton shows only on *initial* load. Toggle-driven recomputes do NOT show a skeleton (they're sub-100ms and would flicker).

### Empty states

| State | Rendering |
|-------|-----------|
| Cohort resolves to 0 patients | Full-page empty state: centred `lucide Users` icon 48px text-gray-300, `outcomesEmptyCohortTitle` as `text-base font-semibold`, `outcomesEmptyCohortBody` as `text-sm text-gray-500`, and an `outcomesEmptyCohortAction` text-link (blue-600 underline) back to `/cohort`. |
| Cohort has patients but no visus observations | Same layout, uses `outcomesNoVisus*` keys. |
| Single panel has 0 qualifying patients (e.g. all patients lack OS data) | Panel-scoped empty — header stays, body shows `outcomesPanelEmpty` centred in text-gray-400. |

| Export CSV button | `button`, focusable; success state announces "CSV exportiert" via `aria-live="polite"` status region |
| Cohort link in empty state | `<a>` or `<Link>`, not `<button>` (it is navigation) |

Focus outline: inherit Tailwind default `focus-visible:outline-2 focus-visible:outline-blue-600 focus-visible:outline-offset-2` — apply to all interactive elements that don't already have a visible focus style.

### ARIA live regions

Summary card values recompute on every toggle/filter change. Wrap the entire summary-cards row in a single `aria-live="polite" aria-atomic="false"` region. Rationale: screen readers announce the updated values ("Patienten: 42. Messungen: 310. OD: 156. OS: 154") without being overwhelmed; `polite` avoids interrupting in-progress reading.

Chart panels do NOT announce on recompute — it would flood the announcement queue. Instead, the page title area gets a `role="status"` span that reads e.g. "Diagramme mit 42 Patient:innen neu berechnet" after each recompute; throttled to fire at most once per 500ms.

### Focus management on drawer open/close

- Drawer opens → focus moves to first interactive element (first radio in X-axis group).
- Drawer closes (X button, Escape, or gear toggle) → focus returns to the gear button.
- Focus trap: NO — drawer is non-modal, Tab can escape it into the page charts, which is intentional (user might want to Tab through a scatter dot while drawer is open).
