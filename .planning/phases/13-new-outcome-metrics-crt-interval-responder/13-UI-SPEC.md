### Chart Palette (from `src/components/outcomes/palette.ts`)

| Eye | Hex | WCAG contrast vs #ffffff | Usage |
|-----|-----|--------------------------|-------|
| OD | `#1d4ed8` (blue-700) | 6.70:1 | Median line, IQR band, scatter — OD sub-panel |
| OS | `#b91c1c` (red-700) | 6.47:1 | Median line, IQR band, scatter — OS sub-panel |
| OD+OS | `#6d28d9` (violet-700) | 7.10:1 | Median line, IQR band, scatter — combined sub-panel |

**CRT panels:** Use the same `EYE_COLORS` palette — CRT shares the OD/OS/combined sub-panel pattern.

**Interval histogram:** Use OD+OS combined color `#6d28d9` for combined view; OD `#1d4ed8` for OD-only; OS `#b91c1c` for OS-only. Median reference line: `#94a3b8` (Tailwind slate-400), `strokeDasharray="5 5"`.

**Responder classification:**
- Responder bucket: `#16a34a` (Tailwind green-600) — median trajectory line + bar segment
- Partial bucket: `#ca8a04` (Tailwind yellow-600) — median trajectory line + bar segment
- Non-responder bucket: `#dc2626` (Tailwind red-600) — median trajectory line + bar segment

Responder color rationale: green/yellow/red is the universal clinical outcome convention; all three exceed WCAG AA 3.0:1 graphical threshold against `#ffffff`.

## Interaction Contract

### Metric Selector (D-01)

- **Component:** Inline segmented control (tab strip) — `<nav role="tablist">` with `<button role="tab" aria-selected>` children.
- **Placement:** Below the existing page header (`<header>` with cohort title + settings gear), above `OutcomesSummaryCards`.
- **Active tab indicator:** Bottom border 2px solid `#1d4ed8` (blue-700) + `font-weight: 600`. Inactive tabs: `text-gray-500`, no border.
- **Tab sizes:** `px-4 py-2` (16px horizontal, 8px vertical), `text-sm` (14px).
- **Keyboard:** Arrow keys navigate between tabs (standard tab-strip ARIA pattern); Enter/Space activate focused tab.
- **URL sync:** Selecting a tab writes `?metric=visus|crt|interval|responder` to `searchParams`; reading on mount pre-selects the tab. Default (no param) = `visus`.
- **State reset on switch:** `resetToMetricDefaults(metric)` fires on every tab change, updating yMetric, axisMode, and layers to per-metric defaults (D-02).

### CRT View (D-03)

- **Layout:** Identical to the existing visus layout — 3-column grid (`xl:grid-cols-3`) with OD / OS / OD+OS combined sub-panels.
- **Panel component:** `OutcomesPanel` with added `metric='crt'` prop. Panel card: `bg-white rounded-xl border border-gray-200 p-5`.
- **Y-axis domain (absolute):** `[0, 800]` µm fixed.
- **Y-axis domain (delta/delta_percent):** Data-driven symmetric range (same logic as visus `yDomain()`).
- **Settings drawer:** Shows axis-mode, y-metric, layer toggles, grid-points — same controls as visus. Y-metric labels adapt: "Absolut (µm)" / "Δ (µm zur Baseline)" / "Δ % (prozentuale Änderung)".
- **Server routing:** Passes `metric: 'crt'` to `postAggregate()` when cohort size > threshold.

### Treatment-Interval Histogram (D-04)

- **Layout:** Single-column full-width chart (`width="100%"` ResponsiveContainer, height 320px). Wrapped in the same panel card style (`bg-white rounded-xl border border-gray-200 p-5`).
- **Chart type:** `BarChart` (Recharts `Bar` inside `BarChart`). Fixed 6 bins.
- **Eye selector:** Three-option toggle above the chart — OD / OS / OD+OS combined. Rendered as a `<div role="group">` with three `<button>` children styled as a segmented control (same visual as metric selector tabs but smaller — `px-3 py-1 text-xs`). Active button: solid fill `#6d28d9` (violet-700), white text. Inactive: white background, `border-gray-200` border, `text-gray-700` text.
- **Median reference line:** Horizontal `<ReferenceLine>` at median gap value. Label on right: "Median {n}d" (`text-xs text-gray-500`).
- **Settings drawer:** Shows only the eye selector when interval metric is active. Axis-mode, y-metric, layer toggles, and grid-points controls are hidden.
- **Bar fill:** Uses active eye's `EYE_COLORS` value. Bar `fillOpacity: 0.85`.
- **Tooltip:** On hover over a bar: bin label + count. `text-sm` in existing `OutcomesTooltip` style white card, `border border-gray-200 rounded shadow-md p-2`.

### Responder Classification (D-05)

- **Layout:** Two stacked sections inside a single panel card (`bg-white rounded-xl border border-gray-200 p-5`):
  1. Top: Grouped bar chart (`BarChart`) — OD / OS / combined on x-axis, bars grouped by bucket (responder / partial / non-responder). Height 240px.
  2. Bottom: Trajectory overlay — `OutcomesPanel` with `metric='responder'`, always showing `delta` y-metric, median layer only, 3 colored lines (green/yellow/red). Height 320px.
- **Separator between sections:** `<hr className="my-6 border-gray-100" />`.
- **Responder threshold input:** Shown in `OutcomesSettingsDrawer` when metric is `responder`. Numeric input (`type="number"`, `min=0`, `step=1`, `defaultValue=5`). Style: `w-24 border border-gray-200 rounded px-2 py-1 text-sm`. Helper text below: `text-xs text-gray-500`.
- **Bar colors:** Green `#16a34a`, yellow `#ca8a04`, red `#dc2626`. Legend below chart, `fontSize: 12`.
- **Settings drawer:** Shows responder threshold input only (axis-mode, y-metric, layer toggles, grid-points all hidden).

### Settings Drawer — Metric-Aware Controls

| Active Metric | Visible Controls |
|--------------|-----------------|
| visus | Axis mode, Y metric (logMAR labels), Layer toggles, Grid points |
| crt | Axis mode, Y metric (µm labels), Layer toggles, Grid points |
| interval | Eye selector only |
| responder | Responder threshold input only |

**Drawer behavior:** Existing slide-in animation (`translate-x-0` / `translate-x-full`), Escape closes, focus first element on open — all preserved.

### Loading State (inherits Phase 12 pattern)

- Server computing indicator: `role="status" aria-live="polite"` span with `text-gray-500 text-sm italic`.
- When CRT routes server-side: same `data-testid="outcomes-server-computing"` pattern and `{t('outcomesServerComputingLabel')}` copy ("Computing on server…").

### Empty States (METRIC-01, METRIC-02, METRIC-03)

All empty states use `OutcomesEmptyState`-style layout: icon + title + body + optional action link. Icon: `Users` (Lucide), `w-12 h-12 text-gray-300`. Min height: `min-h-[60vh]`. Title: `text-base font-semibold text-gray-900`. Body: `text-sm text-gray-500 max-w-md`.

New variants to add to `OutcomesEmptyState`:
- `'no-crt'` — for CRT metric with no CRT observations
- `'no-interval'` — for interval metric where no intervals can be computed
- `'no-responder'` — for responder metric where classification is not possible

### URL Deep-link (METRIC-04)

- `?metric=visus` — default (backward-compatible, no change to existing links)
- `?metric=crt` — CRT view
- `?metric=interval` — interval histogram
- `?metric=responder` — responder classification
- Param read by `OutcomesView` via `useSearchParams().get('metric')` (not `AnalysisPage`).
- On metric-tab click: `setSearchParams((p) => { p.set('metric', value); return p; })` — preserves other params (`?cohort=`, `?filter=`).
