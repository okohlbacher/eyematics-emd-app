| Token | Value | Usage                                   |
|-------|-------|-----------------------------------------|

| sm    | 8px   | Compact padding (`p-2`, `mb-2`)         |
| md    | 16px  | Default element spacing (`space-y-4`)   |
| lg    | 24px  | Card padding (`p-8` = 32px; see note)   |
| xl    | 32px  | Card padding (`p-8`)                    |
| 2xl   | 48px  | Not used on LoginPage                   |
| 3xl   | 64px  | Not used on LoginPage                   |

Exceptions:
- Card uses `p-8` (32px) — matches xl token, within 8-point scale. Source: existing LoginPage.
- Button uses `py-2.5` (10px) — LoginPage existing pattern; preserve exactly for visual consistency.
- Error banner uses `p-3` (12px) — LoginPage existing pattern; preserve exactly.
- Icon touch area: globe icon at `w-3.5 h-3.5` — decorative only, no minimum touch target required.

| Label   | 14px (sm)    | 500 (medium)        | 1.5         |

Source: LoginPage classes — `text-2xl font-bold` (heading), `text-sm font-medium` (label),

## Color

All values extracted from existing LoginPage Tailwind classes. Hex values are Tailwind v4
defaults (no custom palette).

| Role            | Tailwind Class      | Approximate Hex | Usage                                     |
|-----------------|---------------------|-----------------|-------------------------------------------|
| Dominant (60%)  | `bg-white`          | #FFFFFF         | Card surface (login card background)      |
| Secondary (30%) | `from-slate-800 to-slate-900` | #1e293b / #0f172a | Page background gradient    |
| Accent (10%)    | `bg-blue-600`       | #2563EB         | Primary CTA button only                   |
| Destructive     | `bg-red-50 / text-red-700` | #FEF2F2 / #B91C1C | Error banner background and text |

Accent reserved for: **the primary CTA button only** — `bg-blue-600` with `hover:bg-blue-700`.
No other element uses blue-600 as a background. Focus rings (`focus:ring-blue-500`) are
permitted as accessibility affordances on inputs, not classified as accent usage.

Keycloak button: Uses the same `bg-blue-600 text-white` styling as the existing login submit
button — it IS the primary CTA in Keycloak mode, so this is correct.

Info message banner (D-06): Uses `bg-blue-50 border border-blue-200 text-blue-700` —
analogous to the existing error banner pattern but in blue. This is the informational
equivalent of the red error pattern.

## Interaction Contracts

### LoginPage Provider Branch (KC-04, D-05, D-06, D-07)

**Trigger:** `GET /api/auth/config` response includes `provider: 'keycloak'`

**Local mode (provider = 'local') — unchanged:**
- Render existing username/password form (credentials step)
- Render OTP step if server returns `otp_required`
- No change to current behavior

**Keycloak mode (provider = 'keycloak') — new:**
- Replace entire form section with single "Login with Keycloak" button
- Button: full-width, `bg-blue-600 text-white`, `py-2.5 rounded-lg font-medium hover:bg-blue-700 transition-colors` — identical to existing submit button styling
- Subtitle line beneath the "EyeMatics" heading changes to `t('loginKeycloakSubtitle')` ("Single Sign-On")
- On button click: display inline info banner (no navigation, no API call)
  - Banner appears below button
  - Style: `p-3 bg-blue-50 border border-blue-200 rounded-lg text-sm text-blue-700`
  - Icon: `Info` from lucide-react, `w-4 h-4 flex-shrink-0`
  - Content: `t('loginKeycloakInfoTitle')` as bold prefix + `t('loginKeycloakInfoBody')`
- Language toggle remains visible in both modes (existing behavior preserved)

**Loading state:**
- While fetching `/api/auth/config` on mount: render local form as default
  (avoids flash of wrong state; if config fetch fails, fall back to local form silently)
- Provider config is fetched once on mount; no refetch

**API fetch pattern:** Extend existing `twoFactorEnabledRef` fetch in LoginPage
(already calls `/api/auth/config`) to also read `provider` field from the response.
Store in a `providerRef` (or co-locate with existing config ref). No additional
network request.

## Accessibility

- "Login with Keycloak" button: keyboard focusable, `type="button"` (not submit, since
  it shows an info message rather than submitting a form)
- Info banner: appears in DOM flow below button; no ARIA live region required for v1
  (user triggered the click, so they will see the banner)
- Color contrast: `text-blue-700` on `bg-blue-50` meets WCAG AA (contrast ratio ~5.8:1)
- Focus ring: `focus:ring-2 focus:ring-blue-500` on the button — matches existing
  input focus pattern
