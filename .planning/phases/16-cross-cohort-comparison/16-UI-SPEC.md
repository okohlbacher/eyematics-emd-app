## Interaction Contract

### Drawer Behavior

| Interaction | Response |
|-------------|----------|
| Click compare icon | Open `CohortCompareDrawer` (translate-x-0) |
| Click X or press Escape | Close drawer (translate-x-full) |
| Check cohort checkbox | Add cohort ID to `selectedIds`; update `?cohorts=` URL param |
| Uncheck cohort checkbox | Remove from `selectedIds`; update URL |
| Attempt to check 5th cohort | Checkbox is `disabled`; no change |
| Uncheck primary cohort | Not possible — primary cohort checkbox is `disabled checked` |
| Click "Reset to single cohort" | Clear `?cohorts=` param; restore `?cohort=primaryId` |

### URL State Contract

| Param | When present | Format |
|-------|-------------|--------|
| `?cohort=id` | Single-cohort mode | Single ID string |
| `?cohorts=id1,id2` | Cross-cohort mode (2–4 cohorts) | Comma-separated IDs, max 4 |
| Both present | `?cohorts=` takes precedence; `?cohort=` is deleted | Source: CONTEXT.md D-05 |

### Per-Patient Layer in Settings Drawer (cross-cohort mode)

- The `perPatient` checkbox in `OutcomesSettingsDrawer` is hidden (not just disabled) when `isCrossMode` is true.
- A note in its place reads: `t('outcomesComparePerPatientSuppressed')` — "Per-patient lines are suppressed in comparison mode."
- Source: CONTEXT.md D-07, RESEARCH.md Open Question 1.

Legend entry format per cohort (XCOHORT-03):
```
{cohortName} (N={patientCount} patients)
```
This is rendered as the Recharts `<Line name="...">` prop value and appears in the chart legend automatically. No separate i18n key needed — it's data-driven.

Empty state: This phase does not add a new empty state. The existing `OutcomesEmptyState` component handles the no-cohort and all-eyes-filtered cases. Cross-cohort mode with 0 valid cohort IDs degrades silently (unknown IDs skipped; if all skip, renders as single-cohort with `?cohort=` primary).

Error state: No new error state surface. Invalid `?cohorts=` param values (unknown IDs) are silently filtered out — `savedSearches.find()` returns `undefined`, skipped in the memo.

Destructive actions: None in this phase.
