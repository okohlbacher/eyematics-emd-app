**Step 2: Create server/index.ts** — the Express production entry point.

This file must:

1. **Read settings.yaml at startup** (fail fast if missing). Parse with js-yaml. Extract `server.port` (default 3000), `server.host` (default "0.0.0.0"), `server.dataDir` (default "./data"), and `dataSource.blazeUrl`.

2. **Auto-create data directory and seed users.json** (per D-06). If `dataDir` does not exist, create it with `mkdirSync({ recursive: true })`. If `dataDir/users.json` does not exist, write it with the DEFAULT_MANAGED_USERS array (copy verbatim from AuthContext.tsx lines 78-86, with the exact same fields: username, firstName, lastName, role, centers, createdAt).

3. **Create Express app** with `import express from 'express'`.

4. **Mount API handlers FIRST** (per Pattern 4 — route ordering):
   ```typescript
   app.use(issueApiHandler);
   app.use(settingsApiHandler);
   ```
   Import from `./issueApi.js` and `./settingsApi.js` (tsx resolves .ts files from .js imports, matching ESM conventions).

5. **Mount FHIR proxy SECOND** (per Pattern 3 — target derivation):
   ```typescript
   import { createProxyMiddleware } from 'http-proxy-middleware';
   
   function deriveBlazeTarget(blazeUrl: string): string {
     const u = new URL(blazeUrl);
     return `${u.protocol}//${u.host}`;
   }
   
   app.use('/fhir', createProxyMiddleware({
     target: deriveBlazeTarget(blazeUrl),
     changeOrigin: true,
     on: {
       error: (err, _req, res) => {
         console.error('[fhir-proxy] Error:', err.message);
         if (res && 'writeHead' in res) {
           (res as import('http').ServerResponse).writeHead(502, { 'Content-Type': 'application/json' });
           (res as import('http').ServerResponse).end(JSON.stringify({ error: 'FHIR proxy error' }));
         }
       },
     },
   }));
   ```
   CRITICAL: The target must be host-only (e.g., `http://localhost:8080`), NOT the full blazeUrl with `/fhir` path. The `/fhir` mount path handles the prefix.

6. **Mount static file serving THIRD**:
   ```typescript
   app.use(express.static(path.resolve('dist')));
   ```

7. **Mount SPA fallback LAST** (per Pattern 4):
   ```typescript
   app.get('*', (_req, res) => {
     res.sendFile(path.resolve('dist', 'index.html'));
   });
   ```

8. **Start server with minimal logging** (per D-04):
   ```typescript
   app.listen(PORT, HOST, () => {
     console.log(`[server] EMD app running at http://${HOST}:${PORT}`);
     console.log(`[server] FHIR proxy target: ${blazeTarget}`);
     console.log(`[server] Data directory: ${dataDir}`);
   });
   ```

Use `import type { IncomingMessage, ServerResponse } from 'node:http';` if needed for type annotations. Use `import.meta.dirname` or `process.cwd()` for path resolution — be consistent with existing code (process.cwd()).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> Express API | Untrusted HTTP requests hit API handlers (existing auth checks preserved) |
| Express -> Blaze FHIR | Proxy forwards requests to external FHIR server |
| filesystem -> settings.yaml | Config read at startup; no runtime writes to config from this plan |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03 | Spoofing | API handlers | accept | Existing validateAuth() preserved verbatim; no new auth surface in this phase |
| T-01-04 | Info Disclosure | FHIR proxy error handler | mitigate | Error handler logs details server-side only; returns generic 502 to client |
| T-01-05 | Denial of Service | FHIR proxy | accept | No rate limiting in Phase 1; Blaze server handles its own load. Acceptable for internal deployment. |
| T-01-06 | Tampering | settings.yaml | accept | Read at startup only; existing PUT /api/settings has admin-only auth check |
| T-01-07 | Info Disclosure | SPA fallback | mitigate | express.static serves only from dist/; fallback only sends index.html, not arbitrary files |
</threat_model>
