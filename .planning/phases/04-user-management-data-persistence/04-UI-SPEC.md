## Component Inventory

New UI elements introduced by Phase 4 and their visual specification:

### 1. Generated Password Modal (D-02)

Pattern source: FeedbackButton.tsx overlay + RESEARCH.md Pattern 7.

Structure:
```
fixed inset-0 bg-black/30 z-50            (backdrop)
  bg-white rounded-xl p-6 max-w-sm mx-4 shadow-xl   (panel)
    text-sm font-medium text-gray-700 mb-2            (heading: "New user created")
    font-mono text-lg text-center select-all           (password display)
      bg-gray-50 rounded-lg px-4 py-3 border border-gray-200
    flex gap-3 mt-4 justify-end                        (actions row)
      [Copy to clipboard]  [Dismiss]
```

- Copy button: `px-4 py-2 bg-blue-600 text-white text-sm font-medium rounded-lg hover:bg-blue-700`
- After copy: button label changes to "Copied" for 1500ms, then reverts. No separate success state.
- Dismiss button: `px-4 py-2 bg-gray-100 text-gray-700 text-sm font-medium rounded-lg hover:bg-gray-200`
- Modal cannot be closed by clicking the backdrop — admin MUST explicitly dismiss (D-02). No `onClick` on the backdrop layer.
- Escape key does NOT close the modal — security requirement (password must be acknowledged).

### 2. Action Button Loading State (D-04)

When a server request is in-flight (Create User, Delete User), the triggering button:
- Shows a spinner in place of its icon: `animate-spin` on a `Loader2` lucide icon (16px)
- `disabled` attribute set, `opacity-60 cursor-not-allowed` applied
- Button width stays fixed (no layout shift) — use `min-w-[...]` or fixed width

### 3. Data Loading State (D-09)

DataContext parallel fetch shows a loading skeleton while all 4 resources resolve.

Pages that consume persisted data (QualityPage, CohortBuilder saved searches, etc.) render:
- A subtle `animate-pulse bg-gray-100 rounded` placeholder bar in place of the content area
- Height matches the expected content height (e.g., `h-48` for a table area)
- Text: none — visual pulse only, no spinner text

FHIR data area loads independently and is NOT blocked by this loading state.

### 4. Data Error State (D-11)

When DataContext mount fails (server unreachable):
```
bg-red-50 border border-red-200 rounded-lg p-4
  flex items-start gap-3
    AlertCircle w-5 h-5 text-red-500 (flex-shrink-0)
    div
      p.text-sm.font-medium.text-red-800   (heading)
      p.text-sm.text-red-700.mt-1          (body + retry instruction)
      button.mt-3 [Retry]                  (text-sm text-red-700 underline)
```

- Inline within the content area of the affected page — not a full-page takeover
- FHIR data and navigation remain fully usable

### 5. Inline Error Banner (D-10 mutation failures)

When a fire-and-sync mutation fails (e.g., failed to save quality flag, failed to save excluded case):
```
fixed bottom-4 right-4 z-40
  bg-red-600 text-white text-sm rounded-lg px-4 py-3 shadow-lg
    flex items-center gap-2
      X button (dismiss) — w-4 h-4
      error message text
```

- Auto-dismiss after 4000ms if not manually dismissed
- Maximum one banner at a time — new errors replace the current one
- Do NOT use a full toast context system — this is a simple local state in a wrapping component

## Interaction Contracts

### AdminPage: Create User Flow

1. Admin clicks "Add User" button (UserPlus icon + text)
2. Inline form expands below the button (no page navigation, no modal) — matches existing `showForm` pattern
3. Form fields: Username (required), First Name (optional), Last Name (optional), Role (select), Centers (toggle buttons) — NO password field (D-05)
4. Submit button label: "Create User" — shows spinner + disabled during POST
5. On success: form collapses, password modal appears over page
6. On API error (e.g., username taken): form stays open, inline error below the username field (`text-sm text-red-600 mt-1`)
7. After modal dismissed: user list refreshes from GET /api/auth/users (D-04)

### AdminPage: Delete User Flow

1. Admin clicks trash icon on a row
2. The row's action cell shows inline confirm: `"Remove dr.mueller?" [Remove] [Cancel]` — replaces the trash icon in-place
3. Cancel returns to normal trash icon state
4. Remove sends DELETE, button shows spinner during request
5. On success: row disappears (list refreshed from server)
6. Self-delete attempt: trash icon replaced with greyed-out icon with tooltip "Cannot remove your own account" — no confirm step, matches existing `adminNoDelete` pattern

### DataContext: Page-level error vs. inline error

- Mount failure (unreachable server): affected pages show the Data Error State component inline in their main content area (see Component Inventory #4)
- Mutation failure: Inline Error Banner (Component Inventory #5) — does NOT block the page

## Accessibility

Minimum requirements matching existing project patterns:

- All form inputs have associated `<label>` with `for` / `htmlFor` — matches AdminPage.tsx pattern
- Modal role: `role="dialog" aria-modal="true" aria-labelledby` — matches FeedbackButton.tsx pattern
- Generated password modal: `aria-live="polite"` on the password value for screen reader announcement
- Destructive inline confirm: `aria-label="Remove {username}"` on the Remove button
- Loading buttons: `aria-disabled="true"` and `aria-label="Creating user, please wait"` while in-flight
- Error banners: `role="alert"` for screen reader announcement
