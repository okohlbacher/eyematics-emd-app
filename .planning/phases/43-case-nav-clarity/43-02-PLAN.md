<objective>
Make trajectory-plot data points drill down to the corresponding case: a single click
(or keyboard activation) on a scatter point in the Visus/CRT trajectory panel navigates
to that patient's case-detail page, with a pointer-cursor affordance.

Purpose: Closes FALL-010 (drill from a chart data point to the case detail). Reuses the
existing `/case/:caseId` route and `react-router` navigation. Scatter points already
carry `patientId` (= patient pseudonym); the resolution from pseudonym to `PatientCase.id`
happens in OutcomesView against the already-authorized `cohort.cases`.
Output: Clickable/keyboard-accessible scatter points + a navigation handler, plus a test.

Sequencing: depends on 43-01 because both edit src/i18n/translations.ts (43-01 lands its
label keys first; this plan then appends the drill-down hint key). Localization stays
confined to OutcomesPanel (prop) and OutcomesView (handler) so Phase 44's OutcomesView
decomposition (TECH-02) is not fought — the handler is a small, self-contained callback
that moves cleanly into a hook later.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| server→client cohort load | `cohort.cases` come from `/api/fhir/*`, already restricted to the user's authorized centers server-side |
| chart datum → route navigation | clicking a scatter point produces a client-side route change to /case/:id |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-43-03 | Elevation of privilege / IDOR | drill-down navigation | mitigate | Resolve the clicked pseudonym ONLY against `cohort.cases` (the already-authorized, server-restricted set). Navigation is gated on a successful `find`; an unknown/forged pseudonym yields no navigation. The CaseDetailPage itself also resolves against the same authorized `cases` and renders `caseNotFound` for any id outside the set — defense in depth. |
| T-43-04 | Information disclosure | route param | accept | The case id placed in the URL belongs to a case already loaded into the user's authorized cohort; no new data crosses a boundary. Case views remain server-side audit-logged via the /api/* requests. |
</threat_model>
