## Auth shim for tests

Tests do NOT mount the real `authMiddleware` (that requires a real JWT). Instead they inject `req.auth` directly via a small middleware, mirroring the pattern used by `tests/auditApi.test.ts`. The handler only reads `req.auth.preferred_username`, `req.auth.centers`, `req.auth.role` — those are the ONLY fields tests need to populate.
