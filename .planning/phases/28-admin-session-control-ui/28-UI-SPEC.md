# Phase 28 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — manual Tailwind CSS (no shadcn) |
| Preset | not applicable |
| Component library | none (custom components, consistent with existing pages) |
| Icon library | lucide-react |
| Font | Inter (sans), JetBrains Mono (data/mono cells) |

Design system source: `src/index.css` — Tailwind v4 CSS-first config with custom `@theme` tokens.
Light/dark mode: class-based via `.dark` on `<html>`, toggled by ThemeContext. All new components must implement `dark:` variants.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), badge padding |
| sm | 8px | Inline label/input gaps (`gap-2`), tight row padding |
| md | 16px | Default element spacing, form field gaps (`gap-4`) |
| lg | 24px | Card internal padding (`p-5` = 20px nearest existing; section padding `space-y-6`) |
| xl | 32px | Page padding (`p-8`) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch/click targets for action buttons (Revoke, Sign out everywhere): minimum 36px height via `px-3 py-2` — consistent with existing AdminPage action buttons.
- Accordion toggle area: minimum 44px click target height via `py-3` on the row.

## Typography

All sizes use `font-sans` (Inter). Data-cell values (key_id, timestamps) use `font-data` (JetBrains Mono, `font-feature-settings: 'tnum'`).

| Role | Size | Weight | Line Height | Tailwind |
|------|------|--------|-------------|----------|
| Body / table cell | 14px | 400 (regular) | 1.5 | `text-sm` |
| Label / column header | 14px | 500 (medium) | 1.5 | `text-sm font-medium` |
| Section heading | 18px | 600 (semibold) | 1.2 | `text-lg font-semibold` |
| Page heading | 24px | 700 (bold) | 1.2 | `text-2xl font-bold` |

Data cell class: `font-data text-sm` — applied to `key_id`, issued-at, last-used, expires-at columns in the session table.

Source: matches existing AdminPage.tsx (`text-2xl font-bold` for page heading, `text-sm` for table rows, `text-lg font-semibold` for card headings).

## Color

Tokens are defined in `src/index.css` `@theme` block. Use CSS custom properties, not raw hex values.

| Role | Light token | Dark token | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `--color-canvas` (oklch 0.985) | `--color-canvas` (oklch 0.18) | Page background (`bg-gray-900` / `min-h-screen` on AdminPage wrapper) |
| Secondary (30%) | `--color-surface` / `bg-white` | `dark:bg-gray-800` | Card panels (`bg-white dark:bg-gray-800 rounded-xl border`) |
| Accent (10%) | `blue-600` | `blue-500` | See reserved list below |
| Destructive | `red-500` / `coral` | `red-400` | Revoke button, error banners |

**Accent reserved for:**
- Accordion expand toggle icon when the row is expanded (ChevronUp active state)
- Sort indicator arrow when that column is active (matches existing `text-blue-600` pattern)
- Focus ring on inputs (`focus:ring-blue-500`)
- Loading spinner (`Loader2` in teal/blue) during session fetch and "Sign out everywhere"

**Destructive (`red`) reserved for:**
- "Revoke" action button per session row
- Error banners (session fetch failure, revoke failure)
- Validation error styling on TTL inputs when constraints are violated

**Secondary semantic colors:**
- `text-gray-500 dark:text-gray-400` — subtitles, placeholder text, empty states
- `text-green-800 dark:text-green-300` — success banners (TTL saved)
- `amber-soft` background — warning or advisory copy (e.g. "TTL changes take effect on next issued token")

Border pattern: `border border-gray-200 dark:border-gray-700` on card panels. `border-b border-gray-100 dark:border-gray-700/60` between session table rows.

## Component Inventory

### 1. Session Accordion Toggle (AdminPage — per user row)

**Placement:** New column appended to the right of the existing user row action buttons, or integrated into the actions cell as a "Sessions" button.

**Collapsed state:**
```
[ChevronDown icon] Sessions   (text-sm text-gray-500, hover:text-gray-900)
```

**Expanded state:**
- Row background: `bg-indigo-50/30 dark:bg-indigo-900/10` (subtle highlight, distinct from edit row which uses blue)
- ChevronUp icon replaces ChevronDown, color: `text-blue-600`
- Session table renders in a `<tr>` spanning all columns, `colSpan={8}` (one extra for the new sessions column)

**Loading state (lazy fetch on expand):**
```
<Loader2 className="w-4 h-4 animate-spin text-gray-400" /> Loading sessions…
```
Text: `text-sm text-gray-400 dark:text-gray-500`, centered in the expanded cell.

### 2. Session Table (inside accordion)

Card wrapper: `bg-gray-50 dark:bg-gray-800/50 rounded-lg border border-gray-200 dark:border-gray-700 p-4 mt-2 mb-3`

**Header row:** "Sign out everywhere" button floated right, section label floated left.

```
[Key icon] Active Sessions (N)      [LogOut icon] Sign out everywhere
```

Section label: `text-sm font-medium text-gray-700 dark:text-gray-300`

"Sign out everywhere" button:

- Loading (during API call): replace text with `<Loader2 className="w-3 h-3 animate-spin" /> Signing out…`
- Disabled while loading: `disabled:opacity-50`

**Table columns:**

| Column | Width | Content | Class |
|--------|-------|---------|-------|
| Device | ~120px | `Key: {last8}` | `font-data text-xs text-gray-600 dark:text-gray-300` |
| Issued | ~140px | localized date + time | `font-data text-xs text-gray-500 dark:text-gray-400` |
| Last used | ~140px | localized date + time | `font-data text-xs text-gray-500 dark:text-gray-400` |
| Expires | ~140px | localized date + time | `font-data text-xs text-gray-500 dark:text-gray-400` |
| Action | 80px | Revoke button | right-aligned |

Table header: `text-xs font-medium text-gray-400 dark:text-gray-500 uppercase tracking-wide border-b border-gray-200 dark:border-gray-700 pb-2`

Row: `border-b border-gray-100 dark:border-gray-700/60 last:border-0 py-2`

**Revoke button (per row):**
- Default: `inline-flex items-center gap-1 px-2 py-1 text-xs font-medium text-red-600 hover:text-red-800 hover:bg-red-50 dark:text-red-400 dark:hover:bg-red-900/20 rounded transition-colors`
- No icon needed (text-only "Revoke" matches the compact row size; consistent with AdminPage action economy)
- No confirmation dialog (D-04 — consistent with existing delete-user pattern)
- After API 200: row is removed. The component re-fetches the session list (D-15).

**Empty state (no active sessions for user):**
```
No active sessions.
```
Text: `text-sm text-gray-400 dark:text-gray-500 text-center py-4`
i18n key: `adminNoActiveSessions`

**Error state (session fetch failure):**
```
Could not load sessions. Retry
```
Text: `text-sm text-red-500` with an underlined `Retry` link (`text-red-600 hover:text-red-800 underline ml-1`)
i18n keys: `adminSessionsLoadError`, `retry` (existing)

**Error state (revoke failure):**
Inline error text beneath the table: `text-xs text-red-500 mt-1`
i18n key: `adminRevokeError`

### 3. TTL Configuration Inputs (SettingsPage — auth section)

**Placement:** Append a new subsection inside the existing "Two-Factor Authentication" card (`bg-white dark:bg-gray-800 rounded-xl border`), below the TOTP block, separated by a `<hr className="border-gray-100 dark:border-gray-700" />`.

**Section label:**
```
[Clock icon w-4 h-4] Session Token Lifetimes
```
`text-sm font-semibold text-gray-700 dark:text-gray-300 flex items-center gap-2`

**Input pair layout:** `grid grid-cols-1 sm:grid-cols-2 gap-4`

Each input:
```
<label>  Refresh Token TTL (hours)
<input type="number" min="1" />
<p>  hint text
```

Label: `block text-sm font-medium text-gray-700 dark:text-gray-300 mb-1`
Input (default): `w-full border border-gray-300 dark:border-gray-600 dark:bg-gray-700 dark:text-white rounded-lg px-3 py-2 text-sm focus:outline-none focus:ring-2 focus:ring-blue-500`
Input (validation error): append `border-red-400 bg-red-50 dark:bg-red-900/10`
Hint text: `text-xs text-gray-400 dark:text-gray-500 mt-0.5`

**Validation error message** (inline, below input):
`<p className="text-xs text-red-500 mt-1">` — shown when constraints fail (D-08)

**Save button:**
- Shares existing SettingsPage save pattern — no dedicated save button per field; values are saved on explicit "Save" action consistent with the page's existing `handleSave` → `updateSettings` pattern.
- If the settings page does not have a global save for this section, a dedicated "Save TTL" button is added:
  `inline-flex items-center gap-2 px-4 py-2 bg-blue-600 text-white text-sm font-medium rounded-lg hover:bg-blue-700 disabled:opacity-50`
  With Loader2 spinner during save: `<Loader2 className="w-4 h-4 animate-spin" />`

**Success feedback:** reuses existing `savedBanner` pattern on SettingsPage (green banner, `t('settingsSaved')`).

## Interaction Contract

### Session Accordion

| Trigger | Behavior |
|---------|----------|
| Click user row "Sessions" toggle | Expand accordion → fetch `GET /api/auth/sessions?username=…` → show loading state → render session table or empty/error state |
| Click same toggle again | Collapse accordion, discard fetched state |
| Only one accordion open at a time | Expanding a second user collapses the first (consistent with accordion UX convention) |
| Revoke button click | Immediate API call → row removed on 200 → re-fetch session list → show inline error on failure |
| Sign out everywhere click | Button enters loading state → API call → on 200: session list empties, button returns to default → on failure: error banner below table |

### TTL Inputs

| Trigger | Behavior |
|---------|----------|
| Load SettingsPage | `loadSettings()` → convert `refreshTokenTtlMs / 3_600_000` → populate hour inputs |
| Input change | Client-side validate: refresh ≥ 1; absolute cap ≥ refresh value. Show inline error immediately on blur |
| Save | Convert hours → ms → `updateSettings(...)` → show success banner on 200 → show error banner on failure |
| Invalid values | Disable save button while any validation error is present |

## Accessibility Contract

- All interactive elements have visible focus rings: `focus:outline-none focus:ring-2 focus:ring-blue-500`
- Accordion toggle button: `aria-expanded={boolean}` attribute on the toggle element
- Session table: `<table>` with `<thead>` and `<th scope="col">` for each column header
- Revoke buttons: `aria-label={t('adminRevokeSession') + ' ' + keyId}` to distinguish per-row buttons for screen readers
- "Sign out everywhere" button: `aria-busy={signingOut}` during loading state
- TTL number inputs: `aria-describedby` pointing to the hint `<p>` element
- Error messages: `role="alert"` on error banners (consistent with existing SettingsPage `role="alert"` on `saveError`)

## Integration Constraints

- **i18n completeness test:** The project auto-tests that all keys have both `de` and `en` entries. All 19 new keys listed above must be added to `src/i18n/translations.ts` with both locales.
- **No jest-dom:** RTL assertions in tests use `queryByText().not.toBeNull()` / `.toBeNull()` (CLAUDE.md convention).
- **No confirmation dialogs:** Individual revoke and "sign out everywhere" are no-confirm per D-04/D-05. This matches the AdminPage delete-user precedent.
- **Dark mode:** Every Tailwind class that sets color or background must have a `dark:` counterpart. No light-only classes.
- **Font data class:** Timestamp and key_id cells use `className="font-data text-xs"` — the `font-data` utility is defined in `src/index.css`.
