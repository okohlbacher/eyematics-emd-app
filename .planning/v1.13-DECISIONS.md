## B — decided requirements

### B1 — Time filter scopes the doc-quality metrics (not just population)
- **Ranges:** presets **3m / 6m / 1y / all** + a **custom from/to date picker**.
- **Windowing:** ALL Dokumentationsqualität metrics (completeness, plausibility, per-center scores) computed from **windowed observations only**.
- **Denominator:** completeness = `documented-in-window / expected-in-window`. BOTH numerator and denominator scoped to the window. "Expected" uses the **same method the current full-history completeness already uses, clipped to the window** (keeps windowed/full numbers methodologically consistent). Fixes NF's "smaller range → artificially lower metric" complaint.
- Builds on the A2 hotfix (counts already windowed).

### B2 — "Vollzähligkeit" naming collision  ✅ DONE (commit 84d74d4)
- LandingPage `Datenvollzähligkeit` → **"Einwilligungsquote"** (consent rate).
- DocQuality `completeness` → **"Dokumentations-Vollständigkeit"**.
- ℹ definition tooltip on each (InfoTooltip primitive added).
- Labels/i18n only; computation untouched (that's B1).
- **Carry into B1:** DocQuality still has a separate `dataCompleteness` metric labeled plain "Vollständigkeit" — now visually close to "Dokumentations-Vollständigkeit". B1 should reconcile that label.
