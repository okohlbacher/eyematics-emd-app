### Phase 40 — SavedSearch Hardening + Quality Check Configuration
**Requirements:** F-13 (server-side SavedSearch provenance), QUAL-001 (EMDREQ-QUAL-001).
Because D2 persists the parameter selection in the `SavedSearch`, the server-side hardening and the new persisted field are designed together:
- **F-13 first (server foundation):** server generates `SavedSearch` `id`/`createdAt`, sanitizes `filters` at the API boundary (`server/dataApi.ts`); migration plan + contract tests. NOT behavior-preserving — bounded contract change.
- **QUAL-001 on top:** add the persisted per-subcohort "which parameters to check" field through the hardened/sanitized path; quality review scoped to a chosen cohort/subcohort; reuses Phase 39 thresholds.
**Depends on:** 37, 39.
