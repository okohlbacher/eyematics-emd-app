# v1.17 — Triage of NF UAT round 5 (EMD-v1.16-UAT-User-Response_2026-06-16)

Re-test of v1.16. **Confirmed-fixed:** K1 axis-tooltip removal ("indeed removed"), K1d settings button ("improved"), K-bl1/K-bl2 ("Works"). Everything else came back with regressions or partial fixes. This round is dominated by a **critical finding: the v1.16 canvas-scatter (K2) is broken** — it draws no points AND did not remove the SVG-node wall, so performance is still bad (5th round).

## 🔴 Live-reproduced crux (1440×900, clean load, full 245-case cohort)

Measured in-browser on the current `main`:
- **Scatter points are invisible.** All three scatter canvases exist but the backing store was never sized (still the 300×150 default) and **0 non-transparent pixels** are drawn. The chart shows median + IQR only — no dots. → matches tester "scatter-plots are missing".
- **The SVG node wall is fully intact.** Canvas mode still mounts **4,500 `.recharts-scatter-symbol` layers** (1,500-cap × 3 panels), each rendered as `<g><g><g></g></g></g>` → **~13,500 empty `<g>`, 46,235 total DOM nodes**. Returning `<g/>` from the `<Scatter>` shape does **not** stop Recharts from creating a node per datum.
- **Toggling "Streupunkte" took ~10.4 s.** → matches "performance still poor / unresponsive when loading".

**Conclusion:** the K2 canvas refactor is the worst of both worlds — it suppressed the visible SVG circle but left the per-point node cost AND never painted the canvas. It must be re-done (or replaced), not patched.

## 🔴 Open issues (round 5)

### L1 — Canvas scatter broken: no points + node wall remains (CRITICAL; supersedes K2)
See crux above. Root cause: piggy-backing the canvas on the Recharts per-point `shape` callback keeps the node wall (Recharts wraps every datum), and the box-capture/draw path leaves the canvas blank.
**Fix options (DECISION):**
- **A — Proper canvas (surgical):** stop rendering `<Scatter>` in canvas mode entirely. Compute each point's pixel position from the chart's x/y scales ourselves and draw straight to the canvas. Kills the 13.5k nodes **and** draws visible dots. Keeps Recharts for axes/lines/bands.
- **B — Real canvas/WebGL plot lib for Verläufe:** replace the trajectory chart with a canvas/GL plotting library. Solves perf structurally **and** gives the zoom/pan/box-select the tester asked for (L12) for free. Largest change.
- **C — Revert to SVG scatter:** points visible again but the perf complaint returns. Not recommended.
**Confidence:** high on the diagnosis (measured).

### L2 — Performance still poor on load & settings change (HIGH, 5th round)
Tied to L1's node wall (~10 s measured). Resolving L1-A/B should resolve this; if any residual remains it's the per-patient `<Line>` count + 3-panel synchronous mount.
**Confidence:** high it's still broken.

### L3 — "Kohorten vergleichen" still unusable (HIGH, regression persists)

So beyond perf, the compare selector **locks**: once the pre-selected sub-cohorts are removed, no further selection is possible. Needs a repro of the compare-drawer selection state (likely an empty-selection or disabled-state bug, possibly compounded by the v1.15 split pre-selection logic).
**Confidence:** high it's broken; needs interaction repro.

### L4 — Cohort overlay legend / tooltip / chart-type issues (K3 follow-ups)
- **L4a:** IQR legend entry is **missing in the relative Visus/CRT line (trajectory) plot** (present on other charts, absent here).

**Confidence:** high (explicit). Medium effort (the relative-bucketing infra still exists from v1.15; this re-introduces the relative axis but gated on the toggle).

### L7 — "Offene Kreise" (open circles) regression

### L8 — Interpolated values render off the plotted line

**Confidence:** medium; needs a look at the line `type`/`connectNulls`/dot logic.

### L9 — Annotation column misunderstood — needs a SEPARATE column (K6 redo)
Tester: "Still no column header; status column was renamed to 'Annotation'. Intended was that there is an **additional column on the right** which displays the annotation (e.g. the flagged annotation for Duplikat, or implausible values)."
We misread K6: we **renamed the existing status column** to "Annotation". The tester wanted the **status column kept (with its header)** and a **new, separate Annotation column** on the right showing the flag content (e.g. "Duplikat", implausible-value reason). Revert the rename; add a distinct annotation column.
**Confidence:** high.

### L11 — Dark mode: follow system default + full coverage (NEW, Further Remarks)

**Confidence:** high. (a) small; (b) is an audit across every chart/table.
