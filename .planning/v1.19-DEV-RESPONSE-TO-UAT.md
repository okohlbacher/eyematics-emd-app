## Trajectory (Verläufe)

**You wrote:** "The highlighting should work whenever I hover over a line, not only when I hit an actual measurement." · "Scatter points should also be highlighted when I hover over them — right now there are tooltips only, no highlighting."

**What changed:**
- **N2 — hovered scatter points are now highlighted.** Hovering a point in the cloud emphasises that point (an enlarged, ringed marker) in addition to the single detailed tooltip.
- **N1 — hovering a line emphasises it, but only at the measurement points (kept by design).** We investigated full "hover anywhere along the line": the charting framework (Plotly/WebGL) only reports a hover at actual data points, not on the empty stretch between them. The only way to change that is to pack each line with hundreds of synthetic in-between points — which (a) undoes the performance gain that fixed the earlier "Verläufe is slow" complaint, and (b) would make the tooltip show **made-up values that aren't real measurements**, which we don't want in a clinical view. So line emphasis stays anchored to the measurement points. If you'd like, we can instead add **click-to-select a line** or a **patient picker** to highlight a curve — just say the word.

**Status:** ✅ N2 fixed · ⚪ N1 kept by design (you noted "only change when feasible in this framework").

**You also wrote (new):** "It must be self-evident what y-metric setting is used without opening the settings — add a y-axis label."

**What changed (N9):** every trajectory panel now shows a **y-axis label** for the active metric (e.g. "Δ logMAR", "logMAR", "Netzhautdicke (µm)"), so the plotted quantity is clear without opening Diagramm-Anzeige.

**Status:** ✅ Fixed.

## Cohort overlay — IOD axis + distribution plots

**You wrote:** "For some patients the IOD axis does switch, but the individual dates are not removed (e.g. EM-UKT-0027)." · "The distribution bars are not aligned with the percentage y-axis, so patient vs cohort isn't comparable. When the overlay is active: switch the y-axis fully to percentage; one bar for the patient % per range and one for the cohort-median % per range; tooltips with absolute counts; add an explanation."

**What changed:**
- **N4 — IOD relative axis no longer shows leftover dates.** When the overlay is on, the IOD plot now switches cleanly to the relative "Monate seit Erstvisite" axis with **no residual calendar-date ticks** (verified on EM-UKT-0027).
- **N5 — distribution plots reworked for comparability.** With the overlay on, each histogram now shows, per measurement range, **two bars on one shared percentage axis**: the **patient's %** of their own measurements and the **cohort-median %**. Hovering a bar shows the **absolute counts** (the patient's actual count and the cohort-median count), and an ℹ note explains the plot. Patient and cohort are now directly comparable. (Overlay off keeps the simple count histogram.)

**How to verify:** open EM-UKT-0027 → enable "Kohorten-Referenz anzeigen" → the IOD x-axis reads only relative months; the Visus/CRT distribution plots show paired patient/cohort %-bars on a 0–% axis, with counts on hover.

**Status:** ✅ Fixed (N4 + N5).

## Dark mode

**You wrote:** "(a) make the theme selection a collapsible menu with the three options. (b) The following still don't switch to dark mode: 'Show raw data' collapsibles in Verläufe; the Behandlungsintervall and Responder plots; the OCT-Images view in the Data-Quality case; Anamnese, Befunde, adverse events, Medikation and OCT-Images in the case view."

**What changed:**
- **N7 — the theme selector is now a collapsible menu** with the three options (Hell / Dunkel / System) directly selectable, the active one marked. (The cycling button is retired.)
- **N6 — the remaining surfaces now follow dark mode:** the "Show raw data" preview and the Behandlungsintervall + Responder plots in Verläufe; and in the case view the Anamnese, Befunde, adverse-events, Medikation cards and the OCT-image viewer.

**How to verify:** pick "Dunkel" (or "System" on a dark OS) → open a case and the Verläufe view: every figure, table, collapsible and the OCT viewer render dark.

**Status:** ✅ Fixed (a + b).

## Dokumentationsqualität — "Vollzähligkeit" time-range filter

**You wrote:** "The denominator should be adapted to the registered total in the time range."

**What changed (N8):** the Vollzähligkeit **denominator now follows the time-range filter** as well — both the patient count and the percentage reflect the selected period (with no filter it stays the full registered total, matching the start page).

**How to verify:** Dokumentationsqualität → Zeitraum → "Letztes Jahr": the count and the % both update to the windowed population.

**Status:** ✅ Fixed.

## Cohort overlay — context & selection (new)

**You wrote:** "Include information on the currently selected cohort in the cohort-overlay view of the patient." · "Add the option to select a cohort, containing the patient, for the cohort overlay."

**What changed:**
- **N10 — the active overlay cohort is now named** in the patient header ("Vergleichskohorte: … (N=…)"), so it's clear which peer group the overlay represents.
- **N11 — a cohort selector** lets you choose which cohort (of those that contain this patient) the overlay compares against; it defaults to all patients.

**How to verify:** open a case → next to the overlay toggle, the selector lists the patient's cohorts and the header shows the chosen one's name + N.

**Status:** ✅ Fixed.
