## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-28-01 | Elevation of Privilege | GET/DELETE /api/auth/sessions* | mitigate | Inline `req.auth.role !== 'admin'` guard at the top of all three handlers returns 403; covered by `returns 403 for non-admin` tests in sessionRevoke.test.ts |
| T-28-02 | Information Disclosure | GET /api/auth/sessions | mitigate | Endpoint is admin-only; session ids are UUID jtis (non-sequential, non-enumerable); endpoint never added to PUBLIC_PATHS |
| T-28-03 | Tampering (CSRF) | DELETE /api/auth/sessions* | accept | New DELETE routes follow the existing admin-CRUD pattern (DELETE /users/:username): Bearer-token-only, no requireCsrf. Bearer tokens are not auto-sent cross-site, so CSRF risk is low; consistency with established pattern preferred |
| T-28-04 | Information Disclosure | GET /api/auth/sessions query param | mitigate | `username` validated non-empty; passed only as a bound parameter to a prepared statement (no SQL injection surface); `:id` likewise bound-only |
| T-28-05 | Denial of Service (self) | DELETE /api/auth/sessions | accept | An admin could revoke their own sessions; recoverable by re-login; out of scope per CONTEXT.md |
