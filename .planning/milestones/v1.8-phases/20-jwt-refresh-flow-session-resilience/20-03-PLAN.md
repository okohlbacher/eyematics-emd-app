<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Server response → audit middleware | Status code drives skip decision — must be read AFTER res.on('finish') so middleware sees actual response status |
| Audit row body → audit.db file | REDACT_PATHS gate prevents CSRF tokens / cookie values from being persisted on disk |
| i18n key → audit log UI | Untranslated keys would leak the raw `audit_action_unknown` string — defense-in-depth via mapped keys |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-19 | Denial of Service (audit log flooded by ~80 refreshes/user/12h) | auditMiddleware | mitigate | SKIP_AUDIT_IF_STATUS skips ONLY 200-status refresh (D-19, Pitfall 5 option a); failed refreshes still recorded for security visibility |
| T-20-20 | Information Disclosure (CSRF token persisted in audit row body) | auditMiddleware redactBody | mitigate | /api/auth/refresh + /api/auth/logout added to REDACT_PATHS (D-22); body field redacted by existing redactBody helper |
| T-20-21 | Repudiation (failed refresh attack invisible in audit log) | SKIP_AUDIT_IF_STATUS | mitigate | Skip mechanism is status-conditional — 401/403 paths fall through to logAuditEntry; tested explicitly |
| T-20-22 | Information Disclosure via i18n raw-key leak | describeAction fallback | accept | Existing audit_action_unknown fallback returns a translated string, not the raw method/path; new keys properly added before any production traffic |

</threat_model>
