<objective>
Fix Phase 17 UAT Gap #1: dark mode toggle flips the `dark` class on `<html>` but the
dashboard does not visibly change. Root cause is pre-diagnosed: `src/index.css` uses
`@custom-variant dark ...`, which in Tailwind v4 creates a NEW variant but does NOT
override the built-in `dark` variant (which still resolves to `@media (prefers-color-scheme: dark)`).
The correct directive is `@variant dark ...`, which overrides the built-in variant
so `dark:` utilities respond to the `.dark` class instead of the OS media query.

Purpose: Restore the intended class-based dark mode behavior across the app.
Output: One-line CSS change in `src/index.css`.
</objective>
