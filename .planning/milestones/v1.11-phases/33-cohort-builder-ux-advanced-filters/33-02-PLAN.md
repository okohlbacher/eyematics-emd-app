<objective>
Replace the cohort builder's silent input clamping with COH-01 inline validation that blocks
Save, and make filter state survive navigation + reload via sessionStorage (COH-02) with a
working Reset and a logout-clear (D-05). Also teach the two other CohortFilter consumers
(AnalysisPage, OutcomesView) to safe-pick the new Phase 33 fields so they do not silently drop
preset/advanced state (RESEARCH Pitfall 1), and to pass the same getSettings() threshold options
object to applyFilters so preset predicates evaluate against the configured thresholds, not the
fallback defaults.

Depends on Plan 01: the extended CohortFilter type (preset, flaggedCaseIds, advanced fields)
and the applyFilters options parameter must exist before the safe-pick functions can reconstruct
them and before the consumers can pass thresholds.

Purpose: Users see errors instead of silently corrected values, and their filter context is
not lost on navigation — without leaking one user's cohort context to the next on a shared tab.
Output: validated, persisted CohortBuilderPage; logout-clear in AuthContext; updated safe-pick
+ threshold-passing in AnalysisPage + OutcomesView; two new test files.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| sessionStorage <-> CohortBuilderPage | Per-tab client storage; readable/writable by any script in the tab; treated as untrusted on read |
| user keyboard input -> filter fields | Untrusted numeric input crosses here (COH-01 validates it) |
| logout event -> sessionStorage | Security control: persisted filter context must not survive into the next user's session on a shared browser (D-05) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-33-03 | Information Disclosure | emd-cohort-filters persists across users on a shared browser | mitigate | performLogout removes the key (D-05), adjacent to the existing emd-token removal; covered by tests/cohortFilterPersistence.test.tsx logout-clear case. sessionStorage is also tab-isolated and cleared on tab close. |
| T-33-04 | Tampering | corrupt/attacker-written emd-cohort-filters value | mitigate | safePickCohortFilter whitelists every key and validates shape on read (mirrors recentActivityStore); preset accepted only if it is one of the four literals; JSON.parse wrapped in try/catch returning {}. |
| T-33-05 | Input Validation | negative/NaN/out-of-range numeric filter input | mitigate | COH-01 inline validation blocks Save; invalid field is not applied to applyFilters (D-03). ASVS V5 numeric range validation. |
</threat_model>
