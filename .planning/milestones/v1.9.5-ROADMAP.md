## Milestone Summary

**Key Accomplishments:**

1. **Terminology seed extended (SYNTH-01):** `_seedMap` in `src/services/terminology.ts` grows from 10 to 15 entries covering SNOMED 312903003 (DME), 362098006 (RVO), ICD-10-GM E11, H43.1, T85.8. `audit-bundle-codes.mjs` CI gate confirms 0 unresolvable codes across 6 bundles at every `test:ci` run. Drift-guard test in `tests/audit-bundle-codes.test.ts` enforces `_seedMap`/`EXPECTED_SEED_KEYS` symmetry.

2. **Disease-conditional comorbidity model (SYNTH-02):** `sampleComorbidities(primary, age, rand)` helper emits age-correlated comorbidity Conditions. AMD: age-bucket probabilities 30%/60%/80% yield ≥60% rate in shipped bundles (65% actual). DME: 100% diabetes Condition (`E11.9`/`E10.9`) + ≥40% hypertension (`I10`). RVO: 50% `I10`, 30% `E78.0`. All Conditions use BfArM ICD-10-GM system URL, clinicalStatus=active, onset 1–10y before primary.

3. **HbA1c + age coupling + template differentiation (SYNTH-03):** `emitHbA1c` produces 2–5 LOINC 4548-4 Observations per DME case (baseline 7.5–10.5%, ±1.5% step cap, drift toward 7%, clamped 5–13%). `sampleAge` uses cohort-specific truncated normals: AMD N(75,8)[60,95], DME N(65,8)[50,80], RVO N(68,10)[55,85]. `TEMPLATES` constant encodes per-cohort IVI ranges, CRT baselines, drug CDFs (Bevacizumab/Ranibizumab/Aflibercept/Faricimab/Dexamethasone), bilateral probabilities (AMD 30%, DME 60%, RVO 5%), and visus ranges.

4. **Bundle regeneration + CI verification (SYNTH-04):** 4 synthetic site bundles (Chemnitz, Leipzig, Greifswald, Münster) regenerated atomically in a single `chore` commit (D-11). `verify-bundle-distributions.mjs` asserts: AMD median age ≥70 (actual 74), DME diabetes 100%, AMD comorbidity rate ≥60% (actual 65%), DME HbA1c ≥2 obs/case (actual 100%). Both audit and verify scripts chained into `npm run audit:bundles`, wired into `test:ci`. 682/682 tests pass.

**Key Decisions:**

- `EXPECTED_SEED_KEYS` hand-mirrored in audit script (ESM .mjs cannot import .ts); drift caught by drift-guard test at every CI run
- Whitelist split: `WHITELIST_SYSTEMS` (LOINC, ATC blanket-whitelisted) vs `WHITELIST_KEYS` (specific SNOMED laterality/procedure/method codes)
- AMD ≥60% comorbidity assertable at bundle level only (verifier); unit test uses ≥45% to allow for age-distribution variance before age coupling lands in 26-03
- Reference bundles (Aachen, Tübingen) untouched throughout (D-06 honored)
- Atomic 4-bundle commit (D-11) — prevents partial-regeneration hazard
- `BUNDLE_GLOB` env override pattern (mirrors 26-01) used in both audit and verify scripts for unit-testability without touching `public/data/`

**Tech Debt (Non-Blocking):**

- AMD comorbidity unit test asserts ≥45% (not ≥60%): intentional two-layer design — unit test is pre-age-coupling, verifier enforces the real gate. Documented in VERIFICATION.md.
- HbA1c lower bound 7.5% vs ROADMAP 6.5%: implementation narrowed the range; clinically valid for poorly-controlled diabetes.

**Issues Resolved:**

- `SYNTH-04` checkbox in REQUIREMENTS.md was `[ ]` at audit time (documentation tracking inconsistency); corrected to `[x]` before archiving.
