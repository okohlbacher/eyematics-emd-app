## Overview

**Core Value:** Every user sees only authorized data, with tamper-proof audit trail.

**Goal:** Correct the site roster to the 7 real EyeMatics sites with matching synthetic data, and add cohort-level outcome analysis.

v1.5 replaced the synthetic 5-site roster (UKA/UKB/LMU/UKT/UKM) with the 7 real EyeMatics sites (Aachen, Chemnitz, Dresden, Greifswald, Leipzig, Mainz, Tübingen), built a reproducible deterministic FHIR bundle generator to populate synthetic data for the 5 new sites, migrated user-center assignments, and delivered the Cohort Outcome Trajectories analytics view — a longitudinal visus-over-time / treatment-count plotter with OD/OS/combined panels, live toggles, median overlay with IQR band, custom tooltip, interpolation grid slider, summary cards, expandable data preview with CSV export, empty-state screens, and full DE+EN i18n.

## Milestone Summary

**Key Decisions:**
- **Phase split** mid-milestone (08-03 → Phase 9): Enabled a clean re-planning cycle with fresh CONTEXT, research inherited from Phase 8, and explicit blocking anti-patterns rather than trying to recover a fouled execution in place.
- **Blocking anti-patterns via `.continue-here.md`**: `files_modified` as the edit fence + atomic commits per task. Every Phase 9 executor was forced to acknowledge both before making changes. Zero scope drift in Phase 9 (contrast: 292K lines of drift in the reverted 08-03 attempt).
- **IQR band: single `<Area>` with coordinate-array `baseLine`**: Preserved the intent of the locked CONTEXT decision (single Area, not two-area mask) while fixing a Recharts type incompatibility that Phase 8's research had missed.
- **`flattenToRows` helper local to OutcomesDataPreview.tsx**: Rejected extending Phase 8's `Measurement` type with independent `daysSinceBaseline` + `treatmentIndex` fields; rejected calling `computeCohortTrajectory` twice. Small per-observation duplication accepted in exchange for module-boundary stability.
- **Deterministic synthetic data via Mulberry32 PRNG**: Per-site fixed seeds produce byte-identical bundles on re-generation — enables reproducible load-path tests without committing large regenerated files.
- **No new `/api/*` data endpoints**: OUTCOME-09 enforced — all outcome aggregation runs client-side over `useData().activeCases`. Audit beacon is the only new server touchpoint.

**Issues Resolved:**
- 5-site → 7-site roster migration (SITE-01..07)
- Legacy center IDs (`org-ukb`, `org-lmu`, `org-ukm`) fully removed from code, config, users, tests, and docs
- Users.json automatically migrated on server startup to strip removed centers; users with empty center lists reassigned to `org-uka`
- Synthetic data generator enables adding new sites without hand-authoring FHIR bundles
- Cohort-level longitudinal analysis — previously per-patient only, now cross-cohort trajectories with median + IQR overlay
- Pre-existing `MAX_AUDIT_ENTRIES` build error resolved via the pre-close hygiene sweep (commit `2fa0ad6`)
- Stale legacy bundle files (center-bonn/muenchen/muenster.json, ~291K lines) permanently removed

**Issues Deferred:**
- **Server-side pre-aggregation** for very large cohorts (>1000 patients) — v1.5 computes client-side. Can be added as a server endpoint later if perf warrants.
- **Additional outcome metrics** beyond visus (CRT trajectory, treatment-interval distribution, responder classification) — scoped as future analytics features.
- **Cross-cohort comparison** on the same chart — current scope is single-cohort.
- **Visual UAT spot-checks** — Phase 7 admin filter UI + Phase 9 visual items (chart palette, IQR band, tooltip rendering, empty-state visuals) marked `human_needed` in VERIFICATION.md. Automated signals green; visual confirmation recommended before production cutover.
