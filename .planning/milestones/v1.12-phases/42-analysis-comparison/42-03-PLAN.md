<objective>
ANL-012: The active cohort/filter name is shown in Analysis when a filter is loaded directly (`?filters=`), not only via the saved-search path.

Today AnalysisPage.tsx (~215) renders the cohort name only when `activeSavedSearch` is set. On a direct `?filters=<json>` deep-link there is no displayed name even though the cohort is active. This plan derives a display name for the direct-filters path — using a `?name=` query param when present, otherwise a synthesized label ("Gefilterte Kohorte" plus a short human-readable filter summary).

Purpose: A user arriving via a `?filters=` deep-link sees which cohort/filter is active without first saving the search.
Output: A displayed cohort/filter name on the direct-filters path; saved-search path unchanged.
</objective>

| T-42-06 | Information disclosure | `?name=` reflected in header | mitigate | `name` is rendered as React text content (auto-escaped, no dangerouslySetInnerHTML), trimmed, length-capped. | T-42-07 | Spoofing (misleading name) | derived/supplied display name vs actual filter | accept | A user-supplied `?name=` may not match the actual filter; this is cosmetic and read-only. The case count and charts reflect the real (whitelisted) filters regardless of the label. |
