<interfaces>
From server/fhirApi.ts (the server-side defense — do NOT change its signature):
```typescript
// Filters bundles to ONLY those whose org-id (or Patient.meta.source) is in userCenters.
// userCenters comes from req.auth.centers (the authenticated user's authorized centers).
export function filterBundlesByCenters(bundles: FhirBundle[], userCenters: string[]): FhirBundle[];
```
The client multi-select sends NO center list to the server for /api/fhir/bundles —
the server always filters by req.auth.centers. The multi-select is a CLIENT-SIDE
narrowing of already-authorized data. The regression test must prove that even if a
client attempted to pass an unauthorized center, intersecting with userCenters yields
no extra bundles.

New component contract (define in CenterMultiSelect.tsx):
```typescript
export interface CenterMultiSelectProps {
  options: string[];            // display names (QualityPage) — generic so Analysis can pass labels
  selected: string[];           // currently-selected names; empty array = no filter (all)
  onChange: (next: string[]) => void;
  label?: string;               // optional heading; defaults to t('qualityFilterCenter')
}
export function CenterMultiSelect(props: CenterMultiSelectProps): JSX.Element;
```
