<objective>
Harden the SavedSearch persistence contract (SEC-06 / F-13). The server becomes the owner of SavedSearch provenance: it generates `id` (crypto.randomUUID) and `createdAt` (ISO timestamp) on create, ignoring any client-supplied values, and sanitizes the incoming `filters` object through a shared whitelist so no arbitrary object is ever persisted. The client stops generating id/createdAt and adopts the canonical record returned by the server.

Purpose: Today the client generates id/createdAt and the server persists a generic `filters` object (only a size cap + caseId center check). That trusts the client to shape provenance and the filter blob — a spoofing/injection surface. F-13 moves provenance and filter shaping server-side.

Output: A shared server-safe sanitizer, a hardened POST `/saved-searches` endpoint, an updated client save path, and contract tests proving the new behavior.

NOTE — this is a NOT-behavior-preserving contract change. The POST response shape stays the same (`{ savedSearch: {...} }`), but the persisted id/createdAt now differ from what the client sent. Existing rows (created before this change) must continue to load via GET unchanged — only NEW writes get server-owned provenance.
</objective>

## Trust Boundaries

|----------|-------------|
| browser client → POST /api/data/saved-searches | Untrusted JSON body (id, createdAt, filters) crosses into persistence |
| persisted row → GET /api/data/saved-searches → client | Stored JSON re-enters client state |

## STRIDE Threat Register (ASVS L1)

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40-01 | Spoofing | POST /saved-searches `id`/`createdAt` | mitigate | Server generates id (crypto.randomUUID) and createdAt (ISO now); client values ignored entirely (provenance is server-owned) — Task 2 |
| T-40-02 | Tampering | POST /saved-searches `filters` | mitigate | sanitizeSavedSearchFilters whitelist strips every non-known key before JSON.stringify+persist; no arbitrary object stored — Tasks 1+2 |
| T-40-03 | Tampering | filters JSON → prototype pollution | mitigate | Whitelist copy only assigns known own-keys; `__proto__`/`constructor` keys never copied (covered by Task 1 test) |
| T-40-04 | Denial of Service | oversized filters blob | mitigate | 50000-char cap retained, applied to sanitized payload — Task 2 |
| T-40-05 | Elevation of Privilege | caseIds in filters referencing other centers | mitigate | Existing validateCaseCenters check runs against sanitized filters — Task 2 |
| T-40-06 | Information Disclosure | corrupt old-row filters JSON on GET | accept | Existing GET try/catch skips corrupt rows with a warn; no behavior change, low risk (per-user scoped data) |
