- [x] **DATA-GEN-01** — Node.js generator script (`scripts/generate-center-bundle.ts`) produces a valid FHIR R4 Bundle for a given center id/shorthand/patient count. *Validated: Phase 7-02.*

## Future Requirements (Deferred to Later Milestones)

- **Server-side pre-aggregation for very large cohorts (>1000 patients)** — v1.5 computes client-side; a server endpoint can be added later if perf warrants.
- **Additional outcome metrics beyond visus** (CRT trajectory, treatment-interval distribution, responder classification) — future analytics features once OUTCOME-01..12 prove the pattern.
- **Cross-cohort comparison on the same chart** — current scope is single-cohort.

## Out of Scope (explicit)

- Changes to cohort definition/filtering semantics — the Outcomes view consumes cohorts the existing `CohortBuilderPage` already produces.
- Real patient data — synthetic generator produces pseudonymous data only; no PHI pathways are added.
- Statistical inference (p-values, significance testing) — the view is descriptive, not inferential.
- Export to external analysis tools beyond CSV (no SPSS/R/SAS formats).
