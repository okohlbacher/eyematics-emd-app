# Milestone v1.7: Security, Performance & Cross-Cohort

**Status:** ✅ SHIPPED 2026-04-21
**Phases:** 14–17
**Total Plans:** 16 (3 + 4 + 4 + 5)

## Overview

Hardened production security (JWT algorithm pin, auto-generated cohort hash secret, forced password change on default credential, per-user TOTP 2FA with recovery codes), improved runtime performance (O(N+M) case extraction, startup FHIR bundle cache warming), added cross-cohort comparison (up to 4 cohorts overlaid with spaghetti-plot visual hierarchy), upgraded the audit log UI with multi-dimensional filters, and introduced dark-mode infrastructure with WCAG-compliant chart palettes. Phase 18 (Keycloak OIDC Redirect) was deferred — infrastructure is prepared in Phase 6 (v1.0) but requires a real Keycloak instance to verify.

## Phases

### Phase 14: Security Quick Wins & Performance

**Goal**: Production deployments are protected against JWT algorithm confusion and weak secrets; runtime performance is improved and all chart containers are accessible
**Depends on**: Phase 13 (v1.6)
**Requirements**: SEC-01, SEC-02, SEC-03, PERF-01, PERF-02, A11Y-01
**Plans**: 3

Plans:

- [x] 14-01-PLAN.md — JWT algorithm pin (SEC-01) + cohort hash secret auto-generation (SEC-02)
- [x] 14-02-PLAN.md — O(N+M) patient case extraction (PERF-01) + FHIR bundle cache warming (PERF-02)
- [x] 14-03-PLAN.md — Forced password change enforcement backend + frontend (SEC-03) + ARIA chart labels (A11Y-01)

**Details:**
All `jwt.verify()` call sites in `authMiddleware.ts` and `authApi.ts` pin `{ algorithms: ['HS256'] }`. `data/cohort-hash-secret.txt` auto-created at startup on fresh deployments. Users on default `changeme2025!` password redirected to forced-change flow. `extractPatientCases` uses O(N+M) Map pre-grouping. FHIR bundles warmed into `_bundleCache` at startup. Every Recharts trajectory container exposes an `aria-label`.

### Phase 15: TOTP 2FA

**Goal**: Each user account has its own TOTP secret; the shared static OTP code is retired; users locked out of TOTP have a recovery path
**Depends on**: Phase 14
**Requirements**: SEC-04, SEC-05
**Plans**: 4

Plans:

- [x] 15-01-PLAN.md — Scaffolding: otplib+qrcode install, UserRecord extension, expanded authMiddleware/auditMiddleware, Wave 0 test stubs
- [x] 15-02-PLAN.md — Backend: /login TOTP gate + /totp/enroll + /totp/confirm + /verify TOTP&recovery + admin DELETE /users/:username/totp
- [x] 15-03-PLAN.md — Frontend enrollment: TotpEnrollPage + RecoveryCodesPanel + AuthContext state + App.tsx gate + LoginPage OTP length + 21 i18n keys
- [x] 15-04-PLAN.md — AdminPage Reset 2FA button + human end-to-end checkpoint

**Details:**
Users enroll TOTP via QR code (or manual key) with confirmation gate. RFC 6238 ±1-window tolerance replaces the site-wide static `otpCode` for enrolled users. One-time recovery codes shown at enrollment; each burn-on-use. Pre-enrollment users retain static OTP fallback during transition. Admins can reset another user's enrollment from the admin panel.

### Phase 16: Cross-Cohort Comparison

**Goal**: Overlay up to 4 saved cohorts on a single trajectory chart; individual patient curves are visually subordinate to cohort medians
**Depends on**: Phase 13 (v1.6)
**Requirements**: XCOHORT-01, XCOHORT-02, XCOHORT-03, XCOHORT-04, VIS-04
**UI hint**: yes
**Plans**: 4

Plans:

- [x] 16-01-PLAN.md — CohortCompareDrawer + cohort selector UI + ?cohorts= URL param
- [x] 16-02-PLAN.md — OutcomesPanel cross-cohort series rendering + COHORT_PALETTES
- [x] 16-03-PLAN.md — VIS-04 per-patient desaturation in cross-mode + legend with patient counts
- [x] 16-04-PLAN.md — isCrossMode derive + URL restore + integration tests

**Details:**
1–4 saved cohorts render as overlapping series on one `ComposedChart`. Each cohort has a median line + IQR band in its own `COHORT_PALETTES` color (4-color categorical WCAG 3:1+ set, distinct from OD/OS). Legend format: `Cohort A (N=42 patients)`. `?cohorts=id1,id2,...` URL param drives state and restores on reload. In single-cohort mode, per-patient lines are desaturated (#9ca3af, 22%/12% opacity) with the median overplotted at 4px strokeWidth.

## Milestone Summary

**Deferred:**

- Phase 18: Keycloak OIDC Redirect (KEYCLK-01) — postponed; requires a real Keycloak instance; middleware/config prepared in Phase 6 (v1.0). Carry to next milestone.

**Key Decisions:**

- JWT algorithm pinned to `HS256` at every `jwt.verify()` site — defense against alg-confusion attacks
- Cohort hash secret auto-generated on first boot into `data/cohort-hash-secret.txt`; `settings.yaml` fallback removed from read path
- Default-password gate: users on `changeme2025!` are funneled into a forced-change flow before any other route
- Per-user TOTP replaces site-wide static OTP; static OTP retained as pre-enrollment fallback during transition window
- `COHORT_PALETTES`: 4 fixed colors (emerald-700, amber-700, cyan-700, fuchsia-700), WCAG 3:1+ on white, distinct from OD/OS palette
- Cross-mode hierarchy (VIS-04): per-patient lines suppressed; in single-cohort mode desaturated at 22%/12% opacity with 4px median overplot
- `?cohorts=id1,id2,...` URL param drives cross-cohort mode; max 4 silently truncated client-side
- Tailwind v4 dark mode via `@variant dark` (not `@custom-variant`) — class-based strategy
- `DARK_EYE_COLORS` enforced by automated contrast-ratio test against `#111827`
- Theme state: ThemeContext + `localStorage`; System mode listens to `prefers-color-scheme`; FOUC prevented via inline pre-hydration script

**Issues Resolved:**

- C1–C5, H1–H7, M1–M8, L1–L10 findings from v1.7 full-review security pass (see `.planning/reviews/v1.7-full-review/SUMMARY.md`)
- CR-01 (Phase 16): `loginDemoHint` hardcoded credentials gated behind `import.meta.env.DEV`
- UAT gap: `@custom-variant` → `@variant` migration for Tailwind v4 class-based dark mode (17-05)

**Issues Deferred:**

- metricSelector integration tests — `describe.skip` placeholder; needs full router context
- JWT refresh flow — 10-min session hard cap still forces re-login (M6)
- AuditPage useReducer refactor (L4)
- Real Keycloak OIDC redirect flow (KEYCLK-01 / M7 — blocked at initAuth)

**Technical Debt Incurred:**

- Static OTP fallback remains in `/verify` for pre-enrollment users — remove once all users enrolled
- Some pages still read theme via Tailwind `dark:` variants rather than `useTheme()` hook — acceptable for non-chart surfaces
