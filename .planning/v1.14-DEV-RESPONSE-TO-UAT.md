## I1 — Trajectory hover & drill-down (FALL-010 / A1 follow-up)

**You wrote:** "Scatter points are now clickable, but mouseover only highlights one datapoint that is on the same point on the x axis as the mouse, but not the scatter point hovered over." (performance → see I2)

**Root cause:** both the highlight and the click resolved to the point nearest on the X-axis, not the point physically under the cursor — so with several points sharing an x they could highlight/open a different point.

**What changed:** hovering now emphasises the exact point under the cursor, and a click opens *that* point's case (this also removes a latent risk of the drill-down opening the wrong patient when points overlap). The highlight is applied without re-rendering the chart, so it stays cheap on large clouds.

**How to verify:** Analyse → Verläufe → enable "Streupunkte" → move the cursor across points: the point under the cursor enlarges; click it → its case detail opens. With points clustered on the same date, the hovered/clicked point is the correct one.

**Status:** ✅ Fixed.

## I2 — "Verläufe" performance & loading status (A6 / A1 follow-up)

**You wrote:** "Still slow. Potentially render the plots in the background to avoid unresponsive tabs and show system status." / "system is unresponsive while loading."

**Root cause:** on a large cohort the tab painted thousands of per-patient lines + scatter symbols (~14,000 SVG nodes) on the very first render and only hid them afterwards — and there was no progress indicator on this path, so the tab looked frozen.

**What changed:**
- For large cohorts the per-patient curves and scatter are now off **from the first render** (no heavy first paint), still toggleable on demand.
- A "wird berechnet / rendering" status now shows while the tab prepares on the client path — no more silent freeze.
- When you enable scatter on a large cohort it is downsampled (capped per panel, uniform coverage) so it stays responsive; the drill-down/hover still work on every shown point.

**How to verify:** Analyse → Verläufe on a large cohort (e.g. all 245) → the tab opens quickly with no frozen first paint; a brief status indicator appears; switching metric tabs keeps any layer you enabled; turning "Streupunkte" on stays responsive.

**Status:** ✅ Fixed. (A foreground glance to confirm smoothness on your machine is welcome.)

## I3 — Cohort overlay refinements (FALL-011 / A3 follow-up)

**You wrote:** "(1) Both cohort median lines should be in the legend. (2) What are the transparent ranges showing (sdv?)? (3) How is the cohort data aggregated? … by relative timestamps from start of observation … would require relative time on the x axis. (4) The other plots should also be comparable to the cohort."

**What changed:**
- **(1)** Both cohort median lines (Visus + CRT) now appear in the chart legend.
- **(2)** The shaded band is now labelled **"Kohorten-IQR (25.–75. Perzentil)"** — it is the interquartile range (25th–75th percentile), not a standard deviation.
- **(3) + (4)** deferred: aggregating the cohort by **relative time since each patient's first visit** (which implies a relative-time X-axis) and extending the cohort comparison to the **other plots** are larger design changes scheduled for the next round (see Backlog). Today the overlay aggregates by calendar month on the patient's own date axis.

**How to verify:** open a case detail → enable "Kohorten-Referenz anzeigen" → the legend now lists "Kohorten-Median Visus", "Kohorten-Median CRT" and "Kohorten-IQR (25.–75. Perzentil)".

**Status:** 🟡 Partially fixed (legend + band label done; relative-time aggregation and other-plot comparison are in Backlog).

## I5 — "Vollzähligkeit" unified across both tabs (B2 — reversal of the v1.13 change)

**You wrote:** "The opposite was intended: Both tabs should show 'Vollzähligkeit' as number of cases in system / total number of cases (as shown on the landing page). Restore the changes. The tooltips are helpful and should be added to all four metric cards."

**What changed:** the v1.13 rename was reverted. Both the **landing page** and **Dokumentationsqualität** now show **"Vollzähligkeit"** = patients in the system / all registered patients (the landing-page definition, including not-yet-active/stub registrations). It reacts to the time filter coherently — the numerator counts patients active in the selected window while the denominator stays the full registered total, so the value falls for shorter windows rather than collapsing to ~100%. All **four** Dokumentationsqualität cards now carry a definition tooltip.

**How to verify:**
1. Landing page → the full-width card reads **"Datenvollzähligkeit"** (e.g. 14 %, 245 / 1775).
2. Dokumentationsqualität → four cards: **Vollzähligkeit · Vollständigkeit · Plausibilität · Gesamtbewertung**, each with an ℹ tooltip. The aggregate Vollzähligkeit equals the landing figure when unfiltered.
3. Open the Zeitraum filter and narrow it (e.g. "Letzte 3 Monate") → Vollzähligkeit drops (active-in-window / full registered total), it does **not** jump to ~100 %.

**Note:** per your decision, Vollzähligkeit is included in the **Gesamtbewertung** at weight 0.4, so the overall score reflects coverage (≈65 % on the current data).

**Status:** ✅ Fixed.

## I6 — Quality review: annotation persistence + "Bestätigen" tooltip (C1 follow-up)

**You wrote:** "When confirming in line, I lost the information on my label. The Annotation should remain." + "There is no mouseover for 'Bestätigen', but for the other two options." + (backlog) "it looks like we won't be able to correct information in the primary system."

**What changed:**
- A measurement's annotation (the reported error note) now **stays visible after you confirm or resolve** the row — previously it was hidden once the row left the "open" state (the note was never lost, only hidden). System placeholders ("confirmed"/"corrected") are never shown as a note.
- The note is also shown in the read-only audit log.
- The **"Bestätigen"** action now has a mouseover tooltip, like Behoben and Fehler melden.

**How to verify:** Datenqualität → open a case → "Fehler melden" on a row and enter an error type → the note shows on the row → click **Bestätigen** → the row flips to "bestätigt" and the note **still shows**. Confirm a clean row → no placeholder text appears. Hover "Bestätigen" → a tooltip appears.

**Status:** ✅ Fixed. Correcting values directly in the source/primary system remains out of scope for the demonstrator — "Behoben" stays a marker ("im Quellsystem korrigiert") — noted as Backlog.
