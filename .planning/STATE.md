## Accumulated Context

### Inherited from v1.0 / v1.1

- Local vs Blaze bundle discrimination: Organization entry presence. Local per-center files each have one Organization; Blaze synthetic bundles do not.
- Server-side FHIR bundle loading with center filtering is authoritative — client never sees unauthorized center data.
- i18n keys live in a single `src/translations.ts` with `{de, en}` pairs (not split de.ts/en.ts).
- `fhirApiPlugin` validates auth via `validateAuth()` from `utils.ts`; centers read from decoded JWT payload.
- Center configuration loads from `data/centers.json` into `server/constants.ts`; fallback `DEFAULT_CENTERS` exists for when the file is missing or malformed.

### Inherited from v1.5

- 7-site roster (Aachen, Chemnitz, Dresden, Greifswald, Leipzig, Mainz, Tübingen) is authoritative; `data/centers.json` is the single source.
- Synthetic bundles are generated by `scripts/generate-center-bundle.ts` (Mulberry32 PRNG, deterministic per-site seeds).

### Inherited from v1.6

- `shared/` pure-TS module: cohort math shared between server (aggregation) and client (client-path fallback). No imports from `server/` or `src/`.
- Server aggregation at >1000-patient threshold via `POST /api/outcomes/aggregate`. JWT-center-filtered, TTL-cached, byte-identical to client path.
- Cohort IDs HMAC-SHA256 hashed in all audit events (`server/hashCohortId.ts`). Secret in `config/settings.yaml` (`audit.cohortHashSecret`).
- 4 outcome metrics: Visus logMAR, CRT µm, Treatment-Interval histogram, Responder classification. Metric selector with `?metric=` deep-link.
- 430/430 tests passing across 47 files at v1.6 close.
- Visus y-axis domains: absolute [0, 1] logMAR, delta [-1, 1]. CRT: absolute [0, 800] µm, delta data-driven symmetric. Synthetic bundles clamped to [0.1, 1.0] decimal.

### v1.7 Security Context (from cross-AI review, 2026-04-17)

- `server/authMiddleware.ts` `verifyLocalToken`: calls `jwt.verify(token, secret)` without `algorithms` pin — potential alg:none confusion attack.
- `config/settings.yaml` `audit.cohortHashSecret`: defaults to `'please-replace-in-prod'` — should be auto-generated into `data/` like `jwt-secret.txt`.
- `server/authApi.ts` 2FA: uses fixed `otpCode` from settings (shared static secret, not TOTP). Target: RFC 6238 TOTP via `otplib`.
- `server/initAuth.ts` `_migrateUsersJson`: assigns default password `changeme2025!` to records missing `passwordHash`. Should force password change on first login.
- `server/rateLimiting.ts`: in-memory Map resets on restart — low severity.
