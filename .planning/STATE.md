## Accumulated Context

### Inherited from v1.0 / v1.1

- Local vs Blaze bundle discrimination: Organization entry presence. Local per-center files each have one Organization; Blaze synthetic bundles do not.
- Server-side FHIR bundle loading with center filtering is authoritative — client never sees unauthorized center data.
- i18n keys live in a single `src/translations.ts` with `{de, en}` pairs (not split de.ts/en.ts).
- `fhirApiPlugin` validates auth via `validateAuth()` from `utils.ts`; centers read from decoded JWT payload.
- Center configuration loads from `data/centers.json` into `server/constants.ts`; fallback `DEFAULT_CENTERS` exists for when the file is missing or malformed.

### Inherited from v1.5

- 7-site roster (Aachen, Chemnitz, Dresden, Greifswald, Leipzig, Mainz, Tübingen) is authoritative; `data/centers.json` is the single source.
- Synthetic bundles are generated by `scripts/generate-center-bundle.ts` (Mulberry32 PRNG, deterministic per-site seeds).
- Cohort outcome aggregation is **client-side** in `src/pages/OutcomesPage.tsx` + `OutcomesPanel.tsx`, fed by `useData().activeCases`. No `/api/outcomes*` endpoints exist yet — this is the seam Phase 12 pre-aggregation will add.
- OUTCOME-07 (single-source `aggregate` useMemo) and OUTCOME-09 (no new /api/* data endpoints) shipped in v1.5 — server-side pre-aggregation in Phase 12 will extend, not replace, the client path (cohort-size threshold).
- Visual QA items flagged but not functionally blocking: Phase 7 admin filter render (7 sites), Phase 9 chart palette, IQR band, tooltip, empty states.
- Phase 9 code-review INFO findings: IN-01 (audit beacon URL carries cohort id in querystring — PII reduction), IN-02 (data preview row key uses index — stability).
- v1.5 shipped with 313/313 tests passing across 27 files — regression surface for every v1.6 phase.

### v1.6 Architectural Seams

- `src/pages/OutcomesPage.tsx` — route-level page; houses controls + 3 panels; holds `aggregate` useMemo.
- `src/components/outcomes/OutcomesPanel.tsx` — Recharts chart container (median / per-patient / scatter / IQR layers).
- Audit beacon: `GET /api/audit/events/view-open?...` — cohort id currently in querystring (CRREV-01 target).
- `OutcomesDataPreview` — row key currently array index (CRREV-02 target).
- Admin filter UI — on user-management page (VQA-01 locked-to-7-sites snapshot test target).
- `src/translations.ts` + `tests/outcomesI18n.test.ts` — 71-key v1.5 i18n pattern (METRIC-06 extends, VQA-05 adds empty-state keys).
