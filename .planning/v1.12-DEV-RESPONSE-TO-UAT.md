## Closed — please verify

### ✅ #1 — Multi-select centers (QUAL-024)
**Changed:** the single-center dropdown is now a multi-select chip filter, shared by Datenqualität and Analyse. It narrows what you see on the client; the server still restricts to your authorized centers.
**How to verify:**
1. Datenqualität → center filter: click two center chips → only those centers' cases show; deselect all → all your centers show.
2. Repeat on Analyse.
3. (Security) You should never see a center you aren't assigned to, regardless of selection.

### ✅ #4 — Approve/flag control placement (QUAL-025)
**Changed:** the flag-status (approve) control now also appears near the **top** of the case-detail panel, above the full patient values — no more scrolling past everything.
**How to verify:** open a flagged case in Datenqualität → the status dropdown is reachable at the top; changing it there updates the flag (same as the bottom control).

### ✅ #5 — Drill-down from chart to case (FALL-010) — *was a real bug*
**Changed:** you were right that this didn't work — the trajectory scatter points were rendered at zero size (a Recharts v3 issue), so they were invisible and unclickable. Fixed: points now have a visible, clickable size.
**How to verify:** Analyse → Verläufe → the per-visit scatter points are visible; **single-click** a point → it opens that patient's case detail. (Drill-down is intentionally disabled in cohort-compare mode.)

### ✅ #7 — Chart label clarity (FALL-012)
**Changed:** case-detail chart labels are now explicit: CRT legend "CRT (µm)", Visus Y-axis "Visus (Dezimal, bestkorrigiert)", and the interpolation note reads "Offener Kreis = interpolierter Wert (keine Messung)".
**How to verify:** open any case detail with Visus/CRT data → confirm the legend/axis/interpolation wording reads clearly.

### ✅ #9 / AUDIT-02 — Audit log login actor — *your "unauthenticated" question*
**Changed:** you asked why login rows showed `unauthenticated`. Those were *successful* logins (status 200); the username was in the row body but the actor column fell back to `unauthenticated` because a login request has no token yet. Now **login rows show the attempted username** as the actor — so you can see who logged in and, on a failed attempt, which account was targeted. (Password stays redacted; control characters stripped; truly token-less non-login requests still show `unauthenticated`.)
**How to verify:**
1. Log in as `admin` → Protokoll: your login row now shows actor `admin` (not `unauthenticated`).
2. Try a login with a wrong password for some username → the failed row shows that attempted username.
3. PROT-001: deleted users keep their real historical name; they cannot appear as `anonymous` (sessions are revoked on deletion).

## Fixed this round (found via an adversarial self-review) — please re-test

### 🔧 #3 — Absolute counts on the quality overview (QUAL-023)
**Root cause:** you "couldn't find the absolute values" — they were prominent on **Datenqualität** but only the percentage showed on **Dokumentationsqualität** (the count was a tiny footnote). Also, the population total was summing per-center counts, which **double-counted** patients seen at multiple centers.
**Changed:** both quality pages now show a prominent "Grundgesamtheit: N" label, and the count is the number of **distinct patients** (no double-counting), with count/total on each summary card.
**How to verify:** open **Dokumentationsqualität** → a "Grundgesamtheit: N" line is visible above the metric cards, and each card shows an absolute count, not just %.

### 🔧 #6 — Cohort reference overlay in case detail (FALL-011)
**Root cause:** the overlay (a) compared the patient against a cohort band that *included that same patient*, and (b) matched cohort data by exact calendar date — so on real data, where cohort patients rarely have a visit on the patient's exact dates, **it often rendered nothing** (would have looked broken to you).
**Changed:** the index patient is now excluded from its own reference, and cohort statistics are aggregated by **month** so the band/median actually render across the trajectory.
**How to verify:** open a case detail, enable the cohort-reference toggle → a median line + shaded IQR band appear along the Visus/CRT trajectory.

### 🔧 #8 — Missing axis ticks (A-06) — *partially; see open question*
**Changed:** explicit tick configuration added across all case/analysis charts so tick labels no longer drop on tight layouts.
**Re-test + see open question below** (I still need your screenshot to confirm the exact chart you flagged).

### ❓ Q2 — QUAL-022: what should the time filter scope?
Right now the time-range filter on Datenqualität controls **which cases count** toward the Grundgesamtheit (a case is included if it has activity in the window). However, each included case's **quality assessment / anomaly detection still uses its full visit history**, not only the windowed visits.
**Question:** is that the intended semantic ("population = cases active in the window, assessed over their full history"), or should the anomaly checks themselves be restricted to the selected window? I left the assessment over full history (it's the safer clinical default), but it's your call.

### ❓ Q3 — (minor) Center filter with a stale selection
If a center you've selected later disappears from the available list (e.g. data reload), the chip stays selected but matches nothing. Low priority — do you want it auto-pruned, or is it fine as-is?
